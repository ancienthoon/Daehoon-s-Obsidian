## 1. 이진 탐색 트리(BST)란? : "좌우 정렬의 법칙"

이진 탐색 트리는 이진 트리(자식이 최대 2개) 중에서 아주 엄격한 규칙을 가진 트리입니다.

- **왼쪽 자식:** 나(부모)보다 **작은** 값만 올 수 있음.
    
- **오른쪽 자식:** 나(부모)보다 **큰** 값만 올 수 있음.
    

### 💡 중학생 눈높이 비유: "업다운(Up-Down) 게임"

우리가 숫자를 맞출 때 "50보다 커? 작아?"라고 물어보며 범위를 반씩 줄여나가죠? 이진 탐색 트리는 그 **질문들을 나무 모양으로 미리 만들어 놓은 것**과 같습니다.

- 70을 찾고 싶다면? 뿌리(50)보다 크니까 오른쪽으로 가고, 거기 있는 80보다 작으니까 왼쪽으로 가면 바로 찾을 수 있죠!

## 2. 이진 탐색 트리에 포함되는 주요 알고리즘

BST를 유지하고 활용하기 위해 꼭 필요한 4가지 핵심 알고리즘이 있습니다.

### ① 탐색 (Search)

- **원리:** 루트부터 시작해서 찾는 값이 현재 노드보다 작으면 왼쪽, 크면 오른쪽으로 이동합니다.
    
- **속도:** 한 번 내려갈 때마다 후보군이 절반씩 사라지므로 매우 빠릅니다. ($O(\log N)$)
    

### ② 삽입 (Insertion)

- **원리:** 탐색과 똑같이 내려가다가, 더 이상 갈 곳이 없는 빈자리에 "여기가 내 자리구나!" 하고 들어갑니다.
    

### ③ 삭제 (Deletion) - 가장 까다로운 놈!

데이터를 지우고 나서도 "왼쪽 < 부모 < 오른쪽" 규칙이 깨지면 안 됩니다.

- 자식이 없으면? 그냥 지우면 끝.
    
- 자식이 하나면? 자식을 부모 자리로 올리고 지움.
    
- 자식이 둘이면? **오른쪽 자식들 중 가장 작은 놈**을 데려와서 내 자리에 앉히고 지움. (가장 복잡하지만 논리적이죠!)
    

### ④ 순회 (Traversal) - 트리 전체 읽기

나무에 달린 모든 열매를 어떤 순서로 딸 것인가에 대한 알고리즘입니다.

- **중위 순회 (In-order):** 왼쪽 → 나 → 오른쪽 순서로 방문합니다.
    
    - **결과:** BST에서 중위 순회를 하면 신기하게도 데이터가 **크기 순서대로(오름차순)** 출력됩니다!

## 3. C++ 실전 코드 예시

BST의 핵심인 **삽입**과 **탐색**을 구현해 보겠습니다.

```cpp
#include <iostream>

using namespace std;

// 트리의 마디(노드) 구조
struct Node {
    int data;
    Node* left;
    Node* right;
};

// 새로운 노드를 만드는 함수
Node* createNode(int data) {
    Node* newNode = new Node();
    newNode->data = data;
    newNode->left = newNode->right = nullptr;
    return newNode;
}

// 데이터를 규칙에 맞게 집어넣는 함수 (삽입 알고리즘)
Node* insert(Node* root, int data) {
    if (root == nullptr) return createNode(data);

    if (data < root->data) {
        root->left = insert(root->left, data);
    } else {
        root->right = insert(root->right, data);
    }
    return root;
}

// 데이터를 찾는 함수 (탐색 알고리즘)
bool search(Node* root, int target) {
    if (root == nullptr) return false;
    if (root->data == target) return true;

    if (target < root->data) return search(root->left, target);
    else return search(root->right, target);
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);

    Node* root = nullptr;
    root = insert(root, 50);
    insert(root, 30);
    insert(root, 70);
    insert(root, 20);
    insert(root, 40);

    int findMe = 40;
    if (search(root, findMe)) {
        cout << findMe << "을(를) 찾았습니다!\n";
    } else {
        cout << findMe << "이(가) 트리에 없습니다.\n";
    }

    return 0;
}
```

## 4. 주의할 점: "삐딱하게 자란 나무"

BST는 데이터가 들어오는 순서에 따라 모양이 바뀝니다. 만약 `1, 2, 3, 4, 5` 순서대로 넣으면 나무가 오른쪽으로만 길게 자라나서 **그냥 연결 리스트(한 줄 기차)**처럼 되어버립니다.

이러면 $O(\log N)$의 빠른 속도를 잃어버리고 느려지는데($O(N)$), 이를 해결하기 위해 스스로 균형을 맞추는 **AVL 트리**나 **레드-블랙 트리** 같은 심화 개념이 등장하게 됩니다.
