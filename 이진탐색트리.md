## 이진 탐색 트리 (Binary Search Tree, BST) 🌳

---

### 1. 개념과 원리

#### 🤔 BST가 왜 필요할까요?

이런 상황을 상상해 보세요.

> 도서관에 책이 **100만 권** 있어요. 책을 **그냥 쌓아두면** → 원하는 책을 찾으려면 최악의 경우 100만 번 확인해야 해요. 책을 **번호 순서대로 정리해두면** → 중간부터 찾아서 절반씩 줄여나가면 돼요.

이 "절반씩 줄여나가는" 탐색을 **트리 구조**로 구현한 게 BST예요.

---

#### 🧠 핵심 규칙 (딱 하나!)

```
모든 노드에 대해:
  왼쪽 서브트리의 모든 값 < 현재 노드 < 오른쪽 서브트리의 모든 값
```

> 💡 **비유: 업다운 게임** "50을 찾아라" → 루트가 30이면? **오른쪽으로!** "50을 찾아라" → 루트가 70이면? **왼쪽으로!** 매 단계마다 탐색 범위가 **절반**으로 줄어요.

---

#### ⚠️ 힙과 BST의 차이점

||**힙 (Heap)**|**BST**|
|---|---|---|
|목적|최댓값/최솟값 빠르게 꺼내기|빠른 탐색 / 삽입 / 삭제|
|구조|부모 ≥ 자식 (대소 관계만)|왼쪽 < 부모 < 오른쪽|
|중위순회 결과|정렬 안 됨|**오름차순 정렬** ✅|
|탐색 속도|O(N)|**O(log N)** 평균|

|         | 힙                | 이진탐색트리          |
| ------- | ---------------- | --------------- |
| 목적      | 최댓값/최솟값 빠르게 꺼내기  | 빠른 탐색 / 삽입 / 삭제 |
| 구조      | 부모 ≥ 자식 (대소 관계만) | 왼쪽 < 부모 < 오른쪽   |
| 중위순회 결과 | 정렬 안 됨           | **오름차순 정렬** ✅   |
| 탐색 속도   | $O(N)$           | $O(logn)$       |

### 2. 시각적 구조

#### 🌳 `[50, 30, 70, 10, 40, 60, 80]` 순서로 삽입

```
삽입 순서: 50 → 30 → 70 → 10 → 40 → 60 → 80

            [50]
           /    \
        [30]    [70]
        /  \    /  \
      [10][40][60] [80]

규칙 확인:
  50 기준 → 왼쪽(30, 10, 40) 모두 < 50 ✅
           → 오른쪽(70, 60, 80) 모두 > 50 ✅
  30 기준 → 왼쪽(10) < 30 < 오른쪽(40) ✅
```

---

#### 🔍 탐색 동작 — `search(40)` 과정

```
① 루트(50) 확인 → 40 < 50 → 왼쪽으로 ⬅️
② 노드(30) 확인 → 40 > 30 → 오른쪽으로 ➡️
③ 노드(40) 확인 → 40 == 40 → 찾았다! ✅

총 3번만에 탐색 완료 (전체 7개 노드 중)
```

---

#### ➕ 삽입 동작 — `insert(35)` 과정

```
① 루트(50) → 35 < 50 → 왼쪽 ⬅️
② 노드(30) → 35 > 30 → 오른쪽 ➡️
③ 노드(40) → 35 < 40 → 왼쪽 ⬅️
④ NULL 도착 → 여기에 삽입! ✅

            [50]
           /    \
        [30]    [70]
        /  \    /  \
      [10][40][60] [80]
          /
        [35] ← 새로 삽입
```

---

#### ❌ 삭제 동작 — 3가지 경우

```
Case 1: 자식이 없는 노드 삭제 (leaf node) → 그냥 제거
  delete(10):
        [30]           [30]
        /  \    →         \
      [10][40]           [40]

Case 2: 자식이 1개인 노드 삭제 → 자식을 올림
  delete(30): (오른쪽 자식만 있다고 가정)
      [50]           [50]
      /       →      /
    [30]           [40]
       \
       [40]

Case 3: 자식이 2개인 노드 삭제 → 오른쪽 서브트리의 최솟값(중위 후계자)으로 교체
  delete(50):
            [50]                [60]
           /    \    →         /    \
        [30]    [70]        [30]    [70]
                /  \                   \
              [60] [80]               [80]
  ↑ 50을 오른쪽 서브트리의 최솟값(60)으로 교체
```

---

#### 🚨 BST의 치명적 약점 — 편향 트리 (Skewed Tree)

```
삽입 순서가 [10, 20, 30, 40, 50] 이면?

[10]
   \
   [20]
      \
      [30]
         \
         [40]
            \
            [50]

트리가 아니라 그냥 연결 리스트!
탐색이 O(log N)이 아니라 O(N)으로 망가짐 💀
→ 이걸 해결한 게 AVL 트리, Red-Black 트리 (나중에 배워요)
→ C++ STL의 map/set이 내부적으로 Red-Black 트리를 사용해요!
```

### 3. C++ STL 사용법

> BST를 직접 쓰기보다 **균형을 자동으로 맞춰주는 `map`과 `set`** 을 사용해요. 내부 구조가 Red-Black Tree (균형 BST) 이에요.

```cpp
#include <iostream>
#include <set>
#include <map>
using namespace std;

// ✅ set: 중복 없는 정렬된 집합
set<int> s;
s.insert(50);
s.insert(30);
s.insert(70);
s.insert(30);       // 중복 → 무시됨

s.find(30);         // 해당 원소의 iterator 반환 (없으면 s.end())
s.count(30);        // 존재하면 1, 없으면 0
s.erase(30);        // 원소 삭제
s.size();           // 원소 개수

// set 순회 → 자동으로 오름차순!
for (int x : s) cout << x << " ";  // 50 70

// ✅ map: key-value 쌍, key 기준 정렬
map<string, int> m;
m["apple"]  = 3;
m["banana"] = 1;
m["cherry"] = 5;

m.find("apple");    // iterator 반환
m.count("apple");   // 존재하면 1
m.erase("apple");   // 삭제

// map 순회 → key 기준 오름차순!
for (auto& [key, val] : m) {
    cout << key << ": " << val << "\n";
}
// banana: 1
// cherry: 5
```

> 💡 **STL `set` / `map` 핵심 요약**
> 
> |함수|설명|시간복잡도|
> |---|---|---|
> |`insert(x)`|원소 삽입|O(log N)|
> |`erase(x)`|원소 삭제|O(log N)|
> |`find(x)`|원소 탐색|O(log N)|
> |`count(x)`|존재 여부 확인|O(log N)|
> |`size()`|크기 반환|O(1)|

### 4. 직접 구현 (포인터 기반 BST)

```cpp
#include <iostream>
using namespace std;

// ✅ 노드 구조체
struct Node {
    int val;
    Node* left;
    Node* right;
    Node(int v) : val(v), left(nullptr), right(nullptr) {}
};

// ✅ 삽입
Node* insert(Node* root, int val) {
    if (root == nullptr) return new Node(val);  // 빈 자리 → 삽입

    if (val < root->val)
        root->left  = insert(root->left,  val);  // 왼쪽으로
    else if (val > root->val)
        root->right = insert(root->right, val);  // 오른쪽으로
    // val == root->val 이면 중복 → 무시

    return root;
}

// ✅ 탐색
bool search(Node* root, int val) {
    if (root == nullptr) return false;  // 못 찾음
    if (root->val == val) return true;  // 찾음!

    if (val < root->val) return search(root->left,  val);
    else                 return search(root->right, val);
}

// ✅ 최솟값 찾기 (삭제에 활용)
Node* findMin(Node* root) {
    while (root->left != nullptr) root = root->left;
    return root;
}

// ✅ 삭제
Node* erase(Node* root, int val) {
    if (root == nullptr) return nullptr;

    if (val < root->val) {
        root->left  = erase(root->left,  val);
    } else if (val > root->val) {
        root->right = erase(root->right, val);
    } else {
        // Case 1: 자식 없음
        if (!root->left && !root->right) {
            delete root;
            return nullptr;
        }
        // Case 2: 자식 1개
        if (!root->left)  { Node* tmp = root->right; delete root; return tmp; }
        if (!root->right) { Node* tmp = root->left;  delete root; return tmp; }

        // Case 3: 자식 2개 → 중위 후계자(오른쪽 서브트리 최솟값)로 교체
        Node* successor = findMin(root->right);
        root->val       = successor->val;
        root->right     = erase(root->right, successor->val);
    }
    return root;
}

// ✅ 중위 순회 (오름차순 출력)
void inorder(Node* root) {
    if (root == nullptr) return;
    inorder(root->left);
    cout << root->val << " ";
    inorder(root->right);
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);

    Node* root = nullptr;
    root = insert(root, 50);
    root = insert(root, 30);
    root = insert(root, 70);
    root = insert(root, 10);
    root = insert(root, 40);
    root = insert(root, 60);
    root = insert(root, 80);

    inorder(root);  // 10 30 40 50 60 70 80 ✅

    cout << "\n탐색 40: " << (search(root, 40) ? "찾음" : "없음") << "\n";
    cout << "탐색 99: " << (search(root, 99) ? "찾음" : "없음") << "\n";

    root = erase(root, 50);  // 루트 삭제
    inorder(root);  // 10 30 40 60 70 80 ✅

    return 0;
}
```

### 5. 시간 복잡도 분석

|연산|평균|최악 (편향 트리)|이유|
|---|---|---|---|
|삽입|**O(log N)**|O(N)|트리 높이만큼 내려감|
|삭제|**O(log N)**|O(N)|트리 높이만큼 탐색|
|탐색|**O(log N)**|O(N)|절반씩 줄여나감|
|중위순회|**O(N)**|O(N)|모든 노드 방문|

> 💡 **STL `map` / `set`은 왜 항상 O(log N)?** Red-Black Tree로 **균형을 자동 유지**하기 때문에 최악의 경우에도 트리 높이가 O(log N)을 보장해요!

### 6. Codeforces 활용 팁

#### 🏆 BST / set / map이 주로 쓰이는 문제 유형

|유형|설명|
|---|---|
|**중복 제거 + 정렬**|set으로 자동 처리|
|**빈도수 카운팅**|map으로 key-value 관리|
|**범위 탐색**|`lower_bound` / `upper_bound` 활용|
|**순위 / K번째 원소**|ordered set 응용|

#### 📝 추천 연습 문제

|번호|문제명|난이도|포인트|
|---|---|---|---|
|**4C**|Registration System|⭐⭐|map으로 빈도수 카운팅|
|**525E**|Anya and Cubes|⭐⭐|set 탐색 활용|
|**785C**|Anton and Square|⭐⭐⭐|map + 범위 탐색|

> 💡 백준에서 먼저 연습하고 싶다면?
> 
> - **백준 5639** (이진 검색 트리) → BST 직접 구현
> - **백준 7662** (이중 우선순위 큐) → map 응용
> - **백준 1865** → set + lower_bound

### ✅ 2강 핵심 정리

```
BST 규칙   = 왼쪽 < 부모 < 오른쪽 (모든 노드에서 성립)
탐색 원리  = 매 단계마다 탐색 범위가 절반으로 줄어듦 → O(log N)
중위순회   = BST를 중위순회하면 항상 오름차순 정렬
치명적 약점 = 편향 트리 발생 시 O(N)으로 성능 저하
STL 대안   = map / set (내부적으로 균형 BST 사용, 항상 O(log N) 보장)
```
