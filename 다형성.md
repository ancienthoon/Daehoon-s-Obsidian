# 제9장 다형성 (Polymorphism)

---

## 다형성이란?

> **객체 타입이 다르면 똑같은 메시지(함수 호출)에 대해 서로 다르게 동작하는 것**

```cpp
// 같은 speak() 호출이지만
dog.speak();  // → 멍멍
cat.speak();  // → 야옹
```

> 💡 **하나의 코드로 다양한 타입의 객체를 처리** — 객체지향의 핵심 기술

---

## 객체 포인터의 형변환

### 상향 형변환 (Upcasting)

> **자식 → 부모 타입으로 변환** (자식이 부모로 위장)

```cpp
Dog a;
Animal *pa = &a;        // ✅ OK — 부모 포인터가 자식 객체 가리킴
pa = new Dog();         // ✅ OK

Animal a;
Dog *ps = &a;           // ❌ Error — 자식 포인터는 부모 객체 가리킬 수 없음
```

```
부모 포인터 → 자식 객체 가리키기 ✅
자식 포인터 → 부모 객체 가리키기 ❌
```

> ⚠️ 부모로 위장했으니 **부모 멤버만 접근 가능**  
> 자식 멤버 접근하려면 → 하향 형변환 필요

---

### 하향 형변환 (Downcasting)

> **부모 → 자식 타입으로 변환** (부모로 위장한 자식을 다시 자식으로)

```cpp
Shape *ps = new Rectangle();    // 상향 형변환

// 방법 1: 변수로
Rectangle *pr = (Rectangle *) ps;
pr->setWidth(100);

// 방법 2: 한 줄로
((Rectangle *) ps)->setWidth(100);
```

---

### 상향 vs 하향 비교

|구분|상향 (Upcasting)|하향 (Downcasting)|
|---|---|---|
|방향|자식 → 부모|부모 → 자식|
|변환|**자동**|**명시적 캐스팅** 필요|
|안전성|안전|잘못 쓰면 위험|
|접근 범위|부모 멤버만|자식 멤버 접근 가능|

## 가상 함수 (Virtual Function) ⭐

### 가상 함수가 필요한 이유

```cpp
// virtual 없는 경우
Shape *ps = new Rectangle();
ps->draw();   // → "Shape Draw" 출력 (부모 버전 호출)
              //    포인터 타입 기준으로 호출되기 때문
```

```cpp
// virtual 있는 경우
class Shape {
public:
    virtual void draw() { cout << "Shape Draw"; }
};

Shape *ps = new Rectangle();
ps->draw();   // → "Rectangle Draw" 출력 ✅
              //    실제 객체 타입 기준으로 호출
```

---

### 가상 함수 선언

```cpp
class Shape {
public:
    virtual void draw() {       // 부모에 virtual 선언
        cout << "Shape Draw";
    }
};

class Rectangle : public Shape {
public:
    void draw() {               // 자식에서 재정의 (자동으로 virtual)
        cout << "Rectangle Draw";
    }
};

class Circle : public Shape {
public:
    void draw() {
        cout << "Circle Draw";
    }
};
```

> 💡 부모에서 `virtual` 선언하면 → 자식의 동일 함수도 자동으로 가상 함수  
> 자식에 `override` 키워드 붙이면 더 명확 (오타 시 컴파일 에러)

---

### 가상 함수 동작 예시

```cpp
int main() {
    Shape *arrayOfShapes[3];
    arrayOfShapes[0] = new Rectangle();
    arrayOfShapes[1] = new Triangle();
    arrayOfShapes[2] = new Circle();

    for (int i = 0; i < 3; i++)
        arrayOfShapes[i]->draw();   // 각자 다른 draw() 호출!
}
// 출력:
// Rectangle Draw
// Triangle Draw
// Circle Draw
```

> 💡 **다형성의 장점:** 새 도형 클래스 추가해도 `main()`의 루프는 수정 불필요!

---

### 가상 함수 vs 일반 함수

```cpp
// 일반 함수 — 하향 형변환 필요, 변수도 여러 개 필요
Dog* p1 = new Dog();   p1->speak();
Cat* p2 = new Cat();   p2->speak();

// 가상 함수 — 포인터 하나로 모두 처리
Animal* a = new Dog();
a->speak();    // 멍멍
a = new Cat();
a->speak();    // 야옹
```

---

## 정적 바인딩 vs 동적 바인딩

|구분|정적 바인딩|동적 바인딩|
|---|---|---|
|결정 시점|**컴파일** 시간|**실행** 시간|
|속도|빠름|느림|
|대상|일반 함수|**가상 함수**|

```
정적 바인딩: 컴파일 시 어떤 함수 호출할지 이미 결정
동적 바인딩: 실행 중에 실제 객체 타입 보고 함수 결정
             → 가상 함수가 동적 바인딩을 사용
```

---

## 가상 함수 적용 조건

> ⚠️ **포인터 또는 참조자일 때만 가상 함수 동작!**

```cpp
// ✅ 포인터 → 가상 함수 작동
Animal *a = new Dog();
a->speak();    // 멍멍

// ✅ 참조자 → 가상 함수 작동
Dog d;
Animal &a1 = d;
a1.speak();    // 멍멍

// ❌ 일반 객체 → 가상 함수 작동 안 함
Dog d;
Animal a1 = d;
a1.speak();    // Animal speak() (부모 버전 호출)
```

## 참조자와 가상 함수 활용

```cpp
// 부모 타입 매개변수 → 자식 객체도 받을 수 있음
void move(Shape& s, int sx, int sy) {
    s.setOrigin(sx, sy);    // 부모 멤버 호출
    s.draw();               // 가상 함수 → 자식 버전 호출 ✅
}

int main() {
    Rectangle r;
    move(r, 0, 0);    // Rect Draw 출력

    Circle c;
    move(c, 10, 10);  // Circle Draw 출력
}
```

> 💡 **함수 매개변수는 자식보다 부모 타입으로 선언**  
> → 부모뿐만 아니라 모든 자식 객체도 받을 수 있음

## 가상 소멸자

```cpp
// ❌ 가상 소멸자 없는 경우
Animal *a1 = new Dog();
delete a1;
// 출력:
// Animal 소멸자만 호출 → Dog 소멸자 누락 → 메모리 누수!

// ✅ 가상 소멸자 선언
class Animal {
public:
    virtual ~Animal() { cout << "Animal 소멸자"; }
};
// delete 시 Dog 소멸자 → Animal 소멸자 순서로 모두 호출
```

> ⚠️ 부모 포인터로 자식 객체를 `delete`할 때  
> 부모 소멸자에 `virtual` 없으면 자식 소멸자 호출 안 됨 → **메모리 누수**

## 가상 함수 실전 예제 (Employee)

```cpp
class Employee {
    string name; int salary, full_time;
public:
    Employee(string n, int s, int f) : name(n), salary(s), full_time(f) { }
    virtual void display() const {
        cout << name << " " << salary << " " << full_time << endl;
    }
};

class FullTime : public Employee {
    int bonus;
public:
    FullTime(string n, int f, int s, int b) : Employee(n, f, s), bonus(b) { }
    void display() const {
        Employee::display();    // 부모 것 먼저 호출
        cout << bonus << endl;  // 자신의 것만 추가
    }
};

class PartTime : public Employee {
    string EndDate;
public:
    PartTime(string n, int f, int s, string e) : Employee(n, f, s), EndDate(e) { }
    void display() const {
        Employee::display();      // 부모 것 먼저 호출
        cout << EndDate << endl;  // 자신의 것만 추가
    }
};

int main() {
    Employee *a = new FullTime("김철수", 1, 200, 120);
    Employee *b = new PartTime("김현빈", 0, 150, "181230");

    a->display();   // 이름, 정규직, 월급, 보너스 출력
    b->display();   // 이름, 정규직, 월급, 종료일자 출력
}
```

> 💡 공통 부분은 부모 `display()`에서 처리  
> 다른 부분은 자식 `display()`에서 `Employee::display()` 호출 후 추가

## 핵심 정리

```
다형성 = 같은 코드로 다른 객체 동작

조건:
  ① 상속 관계
  ② 부모에 virtual 선언
  ③ 포인터 또는 참조자로 접근

규칙:
  부모 포인터 → 자식 객체 가리키기 ✅
  자식 포인터 → 부모 객체 가리키기 ❌
  가상 함수 → 실제 객체 기준 호출 (동적 바인딩)
  일반 함수 → 포인터 타입 기준 호출 (정적 바인딩)

주의:
  소멸자에도 virtual 필수 (부모 포인터로 delete 시)
  일반 객체 대입 시 가상 함수 동작 안 함 (포인터/참조자만)
  부모 기능은 Employee::display() 로 명시 호출
```
