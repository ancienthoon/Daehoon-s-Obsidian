> [!info] **한 줄 요약**
> 
> 스택은 **LIFO(Last-In, First-Out, 후입선출)** 방식의 자료구조입니다.
> 
> 마치 '프링글스 통'이나 '쌓여있는 접시'처럼, **가장 나중에 넣은 데이터가 가장 먼저 나오는** 구조입니다.

---

## 1. 핵심 기능 (Basic Operations)

- **`push`**: 데이터를 스택의 맨 위에 쌓습니다.
    
- **`pop`**: 스택의 맨 위에 있는 데이터를 꺼냅니다.
    
- **`top`**: 꺼내지는 않고 맨 위에 무엇이 있는지만 확인합니다.
    
- **`empty`**: 스택이 비어있는지 확인합니다.
    
- **시간 복잡도**: 모든 기본 연산은 **$O(1)$**로 매우 빠릅니다.

## 2. C++ 실습 코드
```cpp
#include <iostream>
#include <stack>
#include <string>

using namespace std;

int main() {
    // 입출력 최적화
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    stack<int> s;

    // 데이터 넣기 (push)
    s.push(10);
    s.push(20);
    s.push(30);

    // 맨 위 데이터 확인 (top)
    cout << "현재 top: " << s.top() << "\n"; // 결과: 30

    // 데이터 꺼내기 (pop)
    s.pop(); 
    cout << "pop 후 top: " << s.top() << "\n"; // 결과: 20

    // 비어있는지 확인
    if (!s.empty()) {
        cout << "스택 크기: " << s.size() << "\n";
    }

    return 0;
}
```

## 3. 🎓 편입 면접 예상 질문 & 모범 답안

면접관은 단순히 '정의'를 묻는 것을 넘어 **'원리와 응용'**을 묻습니다. 아래 핵심 포인트 4가지를 꼭 기억하세요.

### Q1. 스택(Stack)과 큐(Queue)의 차이점을 설명해 보세요.

**A:** "스택은 **LIFO(후입선출)** 구조로 가장 나중에 들어온 데이터가 먼저 나가지만, 큐는 **FIFO(선입선출)** 구조로 먼저 들어온 데이터가 먼저 나갑니다. 스택은 주로 '되돌리기(Undo)'나 '함수 호출 관리'에 쓰이고, 큐는 '프로세스 스케줄링'이나 '데이터 대기열'에 주로 사용됩니다."

### Q2. 스택이 실제 컴퓨터 시스템에서 어떻게 활용되나요?

**A:** "대표적으로 **함수 호출 스택(Call Stack)**이 있습니다. 함수가 실행될 때 복귀 주소와 지역 변수를 스택에 쌓고, 실행이 끝나면 pop 하여 이전 위치로 돌아갑니다. 또한 웹 브라우저의 **'뒤로 가기'** 기능이나 에디터의 **'되돌리기(Undo)'** 기능도 스택의 원리를 이용한 것입니다."

### Q3. 재귀 함수(Recursion)와 스택의 관계는 무엇인가요?

**A:** "재귀 함수가 호출될 때마다 컴퓨터 내부의 **시스템 스택**에 함수의 정보가 계속 쌓입니다. 따라서 재귀가 너무 깊어지면 스택 공간이 꽉 차서 **'Stack Overflow'** 에러가 발생하게 됩니다. 모든 재귀 함수는 이론적으로 명시적인 스택 자료구조를 사용하여 반복문 형태로 바꿀 수 있습니다."

### Q4. 스택을 사용할 때 주의할 점은 무엇인가요?

**A:** "비어있는 스택에서 데이터를 꺼내려 하는 **'Stack Underflow'**와, 정해진 메모리 용량을 초과하여 쌓으려는 **'Stack Overflow'**를 주의해야 합니다. 특히 코딩 테스트나 면접에서는 `pop`이나 `top`을 호출하기 전 반드시 `empty` 여부를 먼저 체크하는 방어적 코딩이 중요합니다."

## 🛠️ 스택(Stack) 꼬리물기 질문 대비 전략

> [!warning] **주의 사항** 꼬리물기 질문은 사용자님이 답변한 내용 속의 '단어'에서 시작됩니다. 본인이 확실히 모르는 용어는 답변에 섞지 않는 것이 요령입니다.

### 🌊 꼬리물기 시나리오 1: '재귀와 스택'

- **(질문 1)** 재귀 함수와 스택의 관계를 설명해 보세요.
    
    - **(답변)** 재귀가 호출될 때마다 복귀 주소와 지역 변수가 **시스템 스택(Call Stack)**에 쌓입니다.
        
- **(꼬리 1)** 그럼 재귀가 너무 깊어져서 **Stack Overflow**가 발생하면 어떻게 해결할 건가요?
    
    - **(답변)** 재귀 대신 **반복문(Iteration)**을 사용하거나, 직접 `std::stack` 자료구조를 사용하여 로직을 구현(Iterative approach)하면 힙 메모리를 활용하므로 스택 오버플로우를 피할 수 있습니다.
        
- **(꼬리 2)** 반복문보다 재귀가 갖는 장점은 무엇인가요?
    
    - **(답변)** 알고리즘의 **가독성**이 좋아지고 코드의 **논리적 구조**가 명확해집니다. 특히 트리나 그래프 탐색 시 코드가 훨씬 간결해집니다.
        

---

### 🌊 꼬리물기 시나리오 2: '배열 vs 연결 리스트' (심화)

- **(질문 2)** 스택을 구현할 때 **배열(Array)**과 **연결 리스트(Linked List)** 중 무엇이 더 유리할까요?
    
    - **(답변)** 상황에 따라 다릅니다. 데이터 개수가 정해져 있다면 배열이 빠르고, 가변적이라면 연결 리스트가 유리합니다.
        
- **(꼬리 1)** 배열로 구현했을 때 데이터가 꽉 차면 어떻게 처리할 건가요?
    
    - **(답변)** 기존 배열보다 2배 큰 새로운 배열을 할당하고 기존 데이터를 복사하는 **Dynamic Array(동적 배열)** 방식을 사용합니다. (C++의 `std::vector` 원리)
        
- **(꼬리 2)** 그럼 데이터를 복사하는 시간 복잡도는 $O(n)$인데, 여전히 `push`가 $O(1)$이라고 할 수 있나요?
    
    - **(답변)** 매번 발생하는 일이 아니므로, 전체 연산 횟수로 나누어 계산하는 **분할 상환 시간 복잡도(Amortized Time Complexity)** 관점에서 여전히 $O(1)$입니다.

### 🌊 꼬리물기 시나리오 3: '실무 응용'

- **(질문 3)** 스택을 활용한 알고리즘 중 기억나는 것이 있나요?
    
    - **(답변)** 문자열의 괄호 짝 맞추기나, 수식의 **후위 표기법(Postfix)** 계산이 있습니다.
        
- **(꼬리 1)** 후위 표기법을 왜 스택으로 계산해야 하나요? 연산자 우선순위 때문인가요?
    
    - **(답변)** 네, 후위 표기법은 연산자가 피연산자 뒤에 오므로 괄호가 필요 없습니다. 스택에 숫자를 쌓다가 연산자를 만나면 바로 앞의 숫자 두 개를 꺼내 계산하면 되기 때문에 컴퓨터가 처리하기에 가장 효율적입니다.
