## 힙 (Heap)

---

### 1. 개념과 원리

#### 🤔 힙이 왜 필요할까요?

상황을 상상해 보세요.

> 응급실에 환자들이 계속 들어옵니다. 단순히 **먼저 온 순서**대로 치료하면 안 되고, **가장 위급한 환자**부터 치료해야 해요.

이때 필요한 게 바로 **우선순위 큐(Priority Queue)** 입니다. 그리고 이걸 가장 효율적으로 구현한 자료구조가 **힙(Heap)** 이에요.

---

#### 🧠 핵심 개념

힙은 **완전 이진트리** 형태로, 딱 한 가지 규칙만 지켜요.

> **최대 힙(Max-Heap):** 부모 노드 ≥ 자식 노드 **최소 힙(Min-Heap):** 부모 노드 ≤ 자식 노드

즉, **루트(맨 꼭대기)에 항상 최댓값(또는 최솟값)이 있어요.**

> ⚠️ 힙은 "전체 정렬"이 목적이 아니에요! **"지금 당장 가장 큰/작은 값만 빠르게 꺼내는 것"** 이 목적이에요.

---

### 2. 시각적 구조

#### 🌳 최대 힙 예시 — `[50, 30, 40, 10, 20, 35, 15]` 삽입 후

```
완전 이진트리 형태:

            [50]
           /    \
        [30]    [40]
        /  \    /  \
      [10][20][35] [15]

배열로 표현 (인덱스 1부터):
index:  1    2    3    4    5    6    7
value: [50] [30] [40] [10] [20] [35] [15]

핵심 인덱스 관계:
- 부모(i)의 왼쪽 자식  → 2*i
- 부모(i)의 오른쪽 자식 → 2*i + 1
- 자식(i)의 부모       → i/2
```

---

#### ⚙️ 삽입 동작 — `push(45)` 과정

```
① 맨 끝에 추가:
            [50]
           /    \
        [30]    [40]
        /  \    /  \
      [10][20][35] [15]
      /
    [45]  ← 여기 추가 (index 8)

② 부모(30)와 비교 → 45 > 30 이므로 교환 (Heapify Up ↑):
            [50]
           /    \
        [45]    [40]
        /  \    /  \
      [10][20][35] [15]
      /
    [30]

③ 부모(50)와 비교 → 45 < 50 이므로 멈춤 ✅
```

---

#### ⚙️ 삭제 동작 — `pop()` 과정 (루트 제거)

```
① 루트(50) 제거, 맨 마지막 노드(30)를 루트로 이동:
            [30]
           /    \
        [45]    [40]
        /  \    /  \
      [10][20][35] [15]

② 자식 중 더 큰 값(45)과 비교 → 30 < 45 이므로 교환 (Heapify Down ↓):
            [45]
           /    \
        [30]    [40]
        /  \    /  \
      [10][20][35] [15]

③ 자식(10, 20) 모두 30보다 작음 → 멈춤 ✅
```

### 3. C++ STL 사용법

```cpp
#include <queue>
#include <vector>
using namespace std;

// ✅ 최대 힙 (기본값)
priority_queue<int> maxHeap;

// ✅ 최소 힙
priority_queue<int, vector<int>, greater<int>> minHeap;

maxHeap.push(30);   // 삽입
maxHeap.push(50);
maxHeap.push(10);

maxHeap.top();      // 최댓값 확인 (제거 X) → 50
maxHeap.pop();      // 최댓값 제거
maxHeap.size();     // 원소 개수
maxHeap.empty();    // 비어있는지 확인
```

> 💡 **STL `priority_queue` 핵심 요약**
> 
> |함수|설명|시간복잡도|
> |---|---|---|
> |`push(x)`|원소 삽입|O(log N)|
> |`pop()`|최대/최솟값 제거|O(log N)|
> |`top()`|최대/최솟값 확인|O(1)|
> |`size()`|크기 반환|O(1)|
> |`empty()`|비어있는지 확인|O(1)|

### 4. 직접 구현 (배열 기반 최대 힙)

```cpp
#include <iostream>
using namespace std;

ios_base::sync_with_stdio(false);
cin.tie(NULL);
cout.tie(NULL);

const int MAXN = 100005;
int heap[MAXN];  // 1번 인덱스부터 사용
int heapSize = 0;

// ✅ 삽입: 맨 끝에 넣고 위로 올리기
void push(int val) {
    heap[++heapSize] = val;  // 맨 끝에 추가
    int i = heapSize;

    // 부모보다 크면 계속 올라감 (Heapify Up)
    while (i > 1 && heap[i] > heap[i / 2]) {
        swap(heap[i], heap[i / 2]);
        i /= 2;
    }
}

// ✅ 최댓값 확인
int top() {
    return heap[1];
}

// ✅ 삭제: 루트 제거 후 마지막 노드를 루트로, 아래로 내리기
void pop() {
    heap[1] = heap[heapSize--];  // 마지막 노드를 루트로
    int i = 1;

    // 자식 중 더 큰 값과 교환하며 내려감 (Heapify Down)
    while (true) {
        int left  = 2 * i;
        int right = 2 * i + 1;
        int largest = i;

        if (left  <= heapSize && heap[left]  > heap[largest]) largest = left;
        if (right <= heapSize && heap[right] > heap[largest]) largest = right;

        if (largest == i) break;  // 더 이상 내려갈 필요 없음

        swap(heap[i], heap[largest]);
        i = largest;
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);

    push(30); push(50); push(10); push(45); push(20);

    cout << top() << "\n";  // 50
    pop();
    cout << top() << "\n";  // 45
    pop();
    cout << top() << "\n";  // 30

    return 0;
}
```

### 5. 시간 복잡도 분석

|연산|시간복잡도|이유|
|---|---|---|
|삽입 `push`|**O(log N)**|트리 높이만큼 위로 올라감|
|삭제 `pop`|**O(log N)**|트리 높이만큼 아래로 내려감|
|최댓값 확인 `top`|**O(1)**|루트 노드 바로 접근|
|전체 정렬 (힙 정렬)|**O(N log N)**|N번 삽입 or N번 삭제|

> 💡 **왜 O(log N)인가요?** 완전 이진트리의 높이는 항상 **log₂N** 이에요. 삽입/삭제 시 최악의 경우 루트까지 올라가거나 내려가므로 O(log N)이에요.

### 6. Codeforces 활용 팁

#### 🏆 힙이 주로 쓰이는 문제 유형

|유형|설명|
|---|---|
|**그리디 + 우선순위**|매 순간 최댓값/최솟값을 골라야 할 때|
|**다익스트라 최단경로**|가장 가까운 노드 꺼내기|
|**K번째 원소 찾기**|정렬 없이 K번째 최솟값|
|**이벤트 시뮬레이션**|시간 순 처리, 스케줄링|

#### 📝 추천 연습 문제 (Codeforces)

|번호|문제명|난이도|포인트|
|---|---|---|---|
|**1C**|Candies and Two Sisters|⭐|기본 정렬/우선순위 감각|
|**500B**|New Year Permutation|⭐⭐|그리디 + 우선순위 큐|
|**1037C**|Packets|⭐⭐⭐|최소 힙 활용 시뮬레이션|

### ✅ 1강 핵심 정리

```
힙 = 완전 이진트리 + 부모-자식 대소 관계 규칙
목적 = 최댓값 or 최솟값을 O(1)로 확인, O(log N)으로 삽입/삭제
배열로 구현 가능 (인덱스 관계: 부모=i/2, 왼자식=2i, 오른자식=2i+1)
STL = priority_queue<int> (최대힙)
      priority_queue<int, vector<int>, greater<int>> (최소힙)
```
