## 🟦 STEP 0 — 마스터 컨텍스트 프롬프트 (모든 대화 시작 시 반드시 사용)

```
당신은 숙련된 Python 퀀트 개발자입니다.
나는 한국 주식 시장을 대상으로 하는 가상(페이퍼) 트레이딩 봇을 Python으로 개발하려 합니다.

### 프로젝트 전체 목표
- 실제 돈은 사용하지 않는 가상 트레이딩 시스템
- 한국 주식(KRX) 데이터를 기반으로 동작
- 기술적 지표 기반 자동 매매 전략 실행
- 결과를 로그와 시각화로 확인 가능

### 기술 스택 (고정)
- Python 3.11+
- 데이터 수집: pykrx, FinanceDataReader
- 데이터 처리: pandas, numpy
- 기술 지표: pandas-ta
- 시각화: matplotlib, plotly
- 스케줄링: schedule
- 설정 관리: python-dotenv, yaml
- 로깅: Python logging 모듈 (파일 + 콘솔)
- 테스트: pytest

### 프로젝트 폴더 구조 (고정, 절대 변경 금지)

### 코딩 규칙 (모든 코드에 반드시 적용)
1. 모든 함수에 한국어 docstring 작성
2. 타입 힌트(Type Hint) 필수 사용
3. 예외 처리는 구체적인 예외 클래스 사용 (Exception 남발 금지)
4. 매직 넘버 금지 — 모든 수치는 settings.yaml에서 읽기
5. 로깅은 print() 대신 logger 사용
6. 각 모듈은 단일 책임 원칙(SRP) 준수
7. 한국어 주석으로 핵심 로직 설명

### 응답 형식
- 코드 전체를 제공하고 핵심 부분에 한국어 인라인 주석 추가
- 코드 블록 위에 파일 경로를 명시: # 파일: src/data/fetcher.py
- 구현 후 "테스트 방법"을 간단히 안내
- 이전 단계 코드와 충돌하는 부분이 있으면 명시

이 컨텍스트를 항상 기억하고 모든 코드가 이 구조와 규칙에 맞게 작성되도록 하세요.
```

## 🟩 STEP 4 — 매매 전략 & 가상 브로커 프롬프트

```
[STEP 0 컨텍스트 적용, STEP 1-3 완료됨]

STEP 4: 매매 전략과 가상 주문 처리 시스템을 구현해주세요.

### 구현할 파일
1. src/strategy/base.py
2. src/strategy/rsi_macd.py
3. src/broker/paper_broker.py
4. tests/test_strategy.py
5. tests/test_broker.py

### base.py — 전략 추상 클래스
- BaseStrategy 추상 클래스 (ABC 사용)
- 추상 메서드: generate_signal(df) → Signal
- Signal 데이터클래스: ticker, action(BUY/SELL/HOLD), price, reason, confidence(0~1)
- 모든 전략이 반드시 구현해야 할 인터페이스 정의

### rsi_macd.py — RSI + MACD 복합 전략
- RSIMACDStrategy(BaseStrategy) 클래스
- settings.yaml에서 파라미터 읽기

매수 조건 (AND 조건):
  * RSI < 30 (과매도 구간)
  * MACD 골든크로스 발생 (MACD > Signal 전환)
  * 종가 > MA_20 (상승 추세 확인)

매도 조건 (OR 조건):
  * RSI > 70 (과매수 구간)
  * MACD 데드크로스 발생
  * 볼린저밴드 상단 돌파

- 각 신호에 confidence 점수 계산 로직 포함
- 신호 발생 이유를 reason 필드에 한국어로 기록

### paper_broker.py — 가상 주문 처리
- PaperBroker 클래스
- 주문 처리 메서드:
  * execute_order(signal, quantity) → OrderResult
  * get_position(ticker) → Position | None
  * get_all_positions() → list[Position]
- 주문 내역을 JSON 파일로 영속 저장 (results/orders.json)
- 슬리피지 시뮬레이션 (매수 +0.1%, 매도 -0.1%)
- 거래 수수료 시뮬레이션 (0.015%)
- 잔고 부족 시 예외 처리

### 데이터 클래스 정의
- Signal: ticker, action, price, reason, confidence, timestamp
- Position: ticker, quantity, avg_price, current_price, pnl, pnl_rate
- OrderResult: success, order_id, ticker, action, quantity, price, fee, timestamp
```
