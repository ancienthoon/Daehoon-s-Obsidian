## 객체의 일생

```
객체 생성 → 생성자() → 객체 사용 → 소멸자() → 객체 파괴
              ↑                           ↑
         멤버변수 초기화            포인터 멤버변수가
         (항상 필요)               있을 때만 필요
```

> 💡 **비유:** 자동차가 공장에서 나올 때(생성자) 색상/배기량이 세팅되고, 폐차장에 갈 때(소멸자) 뒷처리를 하는 것

## 6.1 생성자 (Constructor)

### 생성자란?

> **객체가 만들어질 때 자동으로 호출되는 멤버 함수**  
> → 멤버변수에 초기값을 넣어주는 역할

### 생성자의 특징 (암기!)

|특징|설명|
|---|---|
|이름|**클래스 이름과 동일**|
|반환값|**없음** (void도 아님, 아예 안 씀)|
|접근|**반드시 public**|
|중복 정의|**가능** (매개변수가 다르면 여러 개 가능)|

```cpp
class Car {
public:
    Car() { }        // ← 이게 생성자 (클래스 이름과 동일, 반환값 없음)
};
```

---

### 디폴트 생성자

> **매개변수가 없는 생성자**

```cpp
class Car {
private:
    int speed;
    int gear;
    string color;
public:
    Car() {             // 디폴트 생성자
        speed = 0;
        gear  = 1;
        color = "white";
    }
};

int main() {
    Car c1;             // 디폴트 생성자 자동 호출
}
```

#### 디폴트 생성자 자동 생성 규칙

```
생성자를 하나도 안 쓰면  → 컴파일러가 빈 디폴트 생성자 자동 생성
매개변수 있는 생성자만 쓰면 → 디폴트 생성자 자동 생성 ❌
                              → Car() { } 를 직접 써야 함!
```

> ⚠️ **주의:** `Car(int s, int g, string c)` 같은 매개변수 생성자만 만들면, `Car c1;` 이런 호출이 에러남!

---

### 매개변수 있는 생성자

```cpp
class Car {
private:
    int speed, gear;
    string color;
public:
    Car(int s, int g, string c) {
        speed = s;
        gear  = g;
        color = c;
    }
};

int main() {
    Car c1(0, 1, "red");    // 매개변수 전달하며 생성
    Car c2(0, 1, "blue");
}
```

---

### 생성자 중복 정의 (Overloading)

```cpp
class Car {
public:
    Car() {                         // ① 디폴트 생성자
        speed = 0; gear = 1; color = "white";
    }
    Car(int s, int g, string c) {   // ② 매개변수 있는 생성자
        speed = s; gear = g; color = c;
    }
};

int main() {
    Car c1;                   // ① 호출
    Car c2(100, 0, "blue");   // ② 호출
}
```

> 💡 두 생성자를 모두 쓰고 싶으면 **둘 다 직접 작성**해야 함

---

### 디폴트 매개변수 생성자

```cpp
// 매개변수에 기본값을 주면 → Car() { } 따로 안 써도 됨
Car(int s=0, int g=1, string c="red") {
    speed = s; gear = g; color = c;
}

int main() {
    Car c1;              // s=0, g=1, c="red" 로 생성
    Car c2(100);         // s=100, g=1, c="red"
    Car c3(100, 2);      // s=100, g=2, c="red"
    Car c4(100, 2, "blue");
}
```

> ⚠️ **주의:** 디폴트 매개변수 생성자와 `Car() {}` 를 동시에 쓰면 `Car c1;` 호출 시 **어느 걸 호출해야 할지 모호해서 에러!**

---

### 생성자 외부 정의

```cpp
class Car {
public:
    Car();              // 클래스 안에는 원형만 선언
};

Car::Car() {            // 클래스 밖에서 정의
    speed = 0;
    gear  = 1;
    color = "white";
}
```

---

### 객체 생성 방법 총정리

```cpp
// 일반 객체
Car c1;                      // ① 디폴트 생성자
// Car c2();                 // ② ❌ 이건 함수 원형 선언으로 해석됨!
Car c3(100, 3, "white");     // ③ 매개변수 생성자
Car c4 = Car(0, 1, "blue");  // ④ 임시 객체 만들어서 대입

// 포인터 객체 (new 사용)
Car *c5 = new Car;           // 디폴트 생성자
Car *c6 = new Car();         // 디폴트 생성자
Car *c7 = new Car(20, 11, "blue");

// 배열 객체
Car arr[3];                  // 디폴트 생성자 3번 호출
Car *arr2 = new Car[3];      // 동적 배열, 디폴트 생성자 3번
```

> ⚠️ `Car c2();` 는 생성자 호출이 아니라 **함수 원형 선언**으로 해석됨! 절대 이렇게 쓰지 말 것

## 6.2 소멸자 (Destructor)

### 소멸자란?

> **객체가 없어질 때 자동으로 호출되는 멤버 함수**  
> 함수 범위를 벗어나면(함수 종료, 블록 종료) 자동 호출

### 소멸자의 특징

|특징|설명|
|---|---|
|이름|`~클래스이름` (틸드 ~ 붙임)|
|반환값|**없음** (void도 아님)|
|접근|**public**|
|매개변수|**없음** (항상)|
|중복 정의|**불가능**|

```cpp
class Car {
public:
    ~Car() {            // 소멸자
        // 뒷정리 코드
    }
};
```

### 소멸자 작성 기준

```
멤버변수가 일반 변수만 있다면  → 소멸자 안 써도 됨 (자동 생성)
멤버변수에 포인터가 있다면     → 소멸자 반드시 직접 작성!
                               → 포인터가 가리키는 메모리 해제 필요
```

```cpp
class Car {
public:
    Car() {
        cout << "생성자 호출" << endl;
        speed = 0; gear = 1; color = "white";
    }
    ~Car() {
        cout << "소멸자 호출" << endl;
        // 포인터 멤버변수 있으면 여기서 delete
    }
};

int main() {
    Car c1;
    return 0;
}
// 출력:
// 생성자 호출
// 소멸자 호출   ← main() 끝나면서 c1 소멸 → 자동 호출
```

### 생성자 vs 소멸자 비교

|구분|생성자|소멸자|
|---|---|---|
|호출 시점|객체 생성 시|객체 소멸 시|
|이름|`클래스명()`|`~클래스명()`|
|매개변수|가능|불가|
|중복 정의|가능|불가|
|자동 생성|안 써도 빈 것 생성|안 써도 빈 것 생성|
|직접 작성|**항상 권장**|포인터 멤버 있을 때만|

## 6.3 멤버 초기화 목록

### 기본 문법

```cpp
// 일반적인 초기화 방식
Car(int s, int g, string c) {
    speed = s;   // 생성자 { } 안에서 대입
    gear  = g;
    color = c;
}

// 멤버 초기화 목록 방식 ← 더 권장!
Car(int s, int g, string c) : speed(s), gear(g), color(c) {
    // 추가로 할 작업이 있으면 여기에
}
```

> 💡 `:` 뒤에 `멤버변수(초기값)` 형태로 나열  
> 멤버변수가 `{ }` 안으로 들어가기 **전에** 초기화됨

---

### 멤버 초기화 목록 vs 생성자 본문 대입 차이

#### 핵심 개념: 멤버변수 생성 시점

```
객체 생성 → 멤버변수 메모리 생성 → 생성자 { } 실행
                ↑                        ↑
         여기서 초기화 가능           여기서도 값 대입 가능
         (멤버 초기화 목록)          (하지만 이미 생성된 후)
```

#### 4가지 케이스 비교

**케이스 1: 생성자 본문에 지역변수 선언 → 멤버변수 초기화 안 됨 ❌**

```cpp
Car::Car() {
    int speed = 10;       // ← 이건 지역변수! 멤버변수 speed 가 아님
    string color = "white";
}
// a.speed 출력하면 쓰레기값
```

**케이스 2: 멤버 초기화 목록 사용 → 올바르게 초기화 ✅**

```cpp
Car::Car() : speed(10), color("red") {
    // { } 진입 전에 speed=10, color="red" 로 초기화됨
}
// a.speed 출력하면 10
```

**케이스 3: 일부만 초기화 목록, 나머지는 지역변수 → 부분 초기화 ⚠️**

```cpp
Car::Car() : color("red") {
    int speed = 100;     // 지역변수 → 멤버변수 speed는 쓰레기값
}
// a.speed 출력하면 쓰레기값
```

**케이스 4: 초기화 목록 + 본문에 지역변수 동시 → 초기화 목록 값 사용 ✅**

```cpp
Car::Car() : speed(10), color("red") {
    int speed(50);       // 이건 지역변수, 멤버변수 speed에 영향 없음
    string color("white");
}
// a.speed 출력하면 10 (초기화 목록의 값)
```

---

### 초기화 목록 규칙 요약

```
멤버 초기화 목록 : 변수(값)   → { } 진입 전에 초기화
생성자 본문 내 변수명(값)     → 지역변수 생성, 멤버변수 아님!
생성자 본문 내 변수명 = 값    → 멤버변수에 대입 (이미 생성된 후)
```

> ⚠️ **자주 하는 실수:** 생성자 `{ }` 안에서 `int speed = 10;` 이렇게 쓰면 멤버변수가 아니라 지역변수!  
> `speed = 10;` 처럼 **타입 없이** 써야 멤버변수에 대입됨

## 헷갈리는 포인트 정리

### 생성자 관련 자동 생성 규칙

```
아무 생성자도 안 쓴 경우:
  → Car() { }  자동 생성

Car(int s, ...) 만 쓴 경우:
  → Car() { }  자동 생성 ❌
  → Car c1;  호출 시 에러!
  → 직접 Car() { } 작성 필요

디폴트 매개변수 생성자 Car(int s=0, ...) 쓴 경우:
  → Car() { }  따로 필요 없음 (혼용하면 모호성 에러)
```

### `Car c2();` 함정

```cpp
Car c2();   // ← 생성자 호출 아님! 함수 원형 선언으로 해석됨
Car c2;     // ← 올바른 디폴트 생성자 호출
```

### 생성자 본문 안 지역변수 vs 멤버변수

```cpp
Car::Car() {
    int speed = 10;   // ← 지역변수 (멤버변수 speed와 별개)
    speed = 10;       // ← 멤버변수 speed에 대입
}
```

### 소멸자 작성 기준 한 줄 정리

```
포인터 멤버변수 없음 → 소멸자 생략 OK
포인터 멤버변수 있음 → 소멸자 필수 (delete 로 메모리 해제)
```

## 🔑 전체 핵심 요약

```cpp
class Car {
private:
    int speed, gear;
    string color;
public:
    // ① 디폴트 생성자 (항상 작성 권장)
    Car() : speed(0), gear(1), color("white") { }

    // ② 매개변수 생성자
    Car(int s, int g, string c) : speed(s), gear(g), color(c) { }

    // ③ 소멸자 (포인터 멤버 없으면 생략 가능)
    ~Car() { }
};
```

```
핵심 암기:
- 생성자: 클래스이름(), public, 반환값 없음, 중복 가능
- 소멸자: ~클래스이름(), public, 반환값 없음, 매개변수 없음, 중복 불가
- 초기화 목록: 생성자() : 멤버(값) { }  → { } 진입 전에 초기화
- Car c2(); → 함수 원형 선언! 생성자 호출 아님!
```
