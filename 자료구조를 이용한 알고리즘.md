대용량 데이터를 다룰 때 단순히 `for` 문을 돌리는 것($O(n)$)과 적절한 자료구조를 사용하는 것($O(\log n)$ 또는 $O(1)$은 하늘과 땅 차이입니다. 탐색과 정렬 효율을 극대화할 수 있는 핵심 자료구조들을 옵시디언 필기용으로 정리해 드립니다.

# 🚀 자료구조를 이용한 시간 복잡도 혁신

> [!abstract] **핵심 아이디어**
> 
> 데이터가 많아질수록 **'정렬된 상태를 유지하는 힘'**과 **'한 번에 절반씩 버리는 능력'**이 수행 시간을 결정합니다.

---

## 1. 탐색의 혁신: Map & Set (이진 탐색 트리)

데이터를 찾을 때 처음부터 끝까지 다 뒤지는 것은 $O(n)$입니다. 하지만 **이진 탐색 트리(Balanced BST)** 기반의 자료구조를 쓰면 탐색 시간을 $O(\log n)$으로 줄일 수 있습니다.

- **원리:** 데이터를 넣을 때마다 크기를 비교해 왼쪽/오른쪽 자식으로 보냅니다. 한 번 비교할 때마다 후보군이 절반으로 줄어듭니다.
    
- **비유:** 100만 페이지짜리 사전에서 '사과'를 찾을 때, 1페이지부터 넘기는 게 아니라 중간을 계속 펼쳐보며 범위를 좁히는 것과 같습니다.
    

### C++ 실전 활용 (`std::map`, `std::set`)

```cpp
#include <iostream>
#include <map>

using namespace std;

int main() 
{
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    map<int, string> m;
    // 삽입 시 자동 정렬 및 트리 구조 형성 (O(log n))
    m[10] = "data1";
    m[5] = "data2";
    m[20] = "data3";

    // 탐색 (O(log n))
    if (m.find(5) != m.end()) {
        cout << "found: " << m[5] << "\n";
    }

    return 0;
}
```

## 2. 극한의 속도: Hash Table (해시 테이블)

만약 "나는 정렬 같은 거 필요 없고, 무조건 찾는 속도가 제일 중요해!"라고 한다면 해시(Hash)가 정답입니다.

- **원리:** 키(Key) 값을 수식(Hash Function)에 넣어 데이터가 저장될 '방 번호'를 바로 계산합니다.
    
- **복잡도:** **$O(1)$** (데이터가 1억 개라도 단 한 번에 찾음).
    
- **비유:** 아파트 호수를 알고 있어서 바로 엘리베이터를 타고 그 집으로 찾아가는 것과 같습니다.
    

### C++ 실전 활용 (`std::unordered_map`)

```cpp
#include <iostream>
#include <unordered_map>

using namespace std;

int main() 
{
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    unordered_map<string, int> hash_table;
    hash_table["apple"] = 1000; // O(1)
    
    // 데이터가 100만 개가 있어도 찾는 속도는 일정함
    cout << "price: " << hash_table["apple"] << "\n";

    return 0;
}
```

## 3. 실시간 정렬 유지: Heap (우선순위 큐)

모든 데이터를 정렬할 필요는 없지만, **"지금 당장 가장 큰 값(혹은 작은 값)"**이 계속 필요한 경우 유용합니다.

- **원리:** 완전 이진 트리의 형태로, 부모 노드가 자식보다 항상 크거나 작도록 유지합니다.
    
- **효율성:** * 전체 정렬: $O(n \log n)$
    
    - 최댓값/최솟값 추출: $O(\log n)$ (정렬된 배열에서 하나를 빼고 다시 정렬하는 $O(n)$보다 훨씬 빠름)
        
- **비유:** 응급실에서 환자가 들어올 때마다 위급한 순서대로 줄을 세우는 시스템입니다.
    

### C++ 실전 활용 (`std::priority_queue`)

```cpp
#include <iostream>
#include <queue>

using namespace std;

int main() 
{
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    // 기본적으로 내림차순 정렬됨
    priority_queue<int> pq;

    pq.push(10); // O(log n)
    pq.push(30);
    pq.push(20);

    // 가장 큰 값 확인 및 제거 (O(log n))
    while (!pq.empty()) 
    {
        cout << pq.top() << " "; // 30, 20, 10 순서로 출력
        pq.pop();
    }

    return 0;
}
```

## 4. 문자열 탐색의 끝판왕: Trie (트라이)

수많은 문자열 속에서 특정 단어를 찾아야 할 때, 문자열 비교는 시간이 오래 걸립니다. 이때 **트라이**를 사용합니다.

- **효율성:** 문자열의 길이가 $L$일 때, 탐색 시간은 데이터 개수와 상관없이 오직 **$O(L)$**입니다.
    
- **비유:** 휴대폰 전화번호부에서 이름을 검색할 때 한 글자씩 입력하면 자동 완성되는 것과 같습니다.

## 🎓 자료구조 선택 가이드 (핵심 요약)

| **상황**                  | **추천 자료구조**                      | **시간 복잡도**  |
| ----------------------- | -------------------------------- | ----------- |
| 단순 존재 여부/값 탐색 (정렬 불필요)  | `unordered_set`, `unordered_map` | $O(1)$      |
| 탐색과 동시에 **항상 정렬 상태** 유지 | `set`, `map` (BST)               | $O(\log n)$ |
| **최댓값/최솟값**만 계속 뽑아야 할 때 | `priority_queue` (Heap)          | $O(\log n)$ |
| 접두사(Prefix) 기반 문자열 탐색   | `Trie`                           | $O(L)$      |
