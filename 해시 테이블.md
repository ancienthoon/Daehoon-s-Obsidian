# 해시 테이블 (Hash Table)

---

## 해시 테이블이란?

> **키(Key)를 해시 함수에 넣어 나온 인덱스에 값을 저장하는 자료구조**

```
Key → [해시 함수] → Index → [배열] → Value

"apple" → h("apple") → 3 → arr[3] = "사과"
```

> 💡 **비유:** 사물함 — 이름표(Key)를 규칙(해시 함수)에 따라 번호(Index)로 바꿔서 해당 칸에 물건 보관

---

## 핵심 개념

### 해시 함수 (Hash Function)

> Key를 Index로 변환하는 함수

```
h(key) = key % 배열크기   ← 가장 단순한 예

h(10) = 10 % 7 = 3   → index 3에 저장
h(17) = 17 % 7 = 3   → 충돌 발생!
```

**좋은 해시 함수의 조건:**

```
✅ 빠르게 계산 가능
✅ 충돌이 적게 발생
✅ 고르게 분산
```

---

## 시간 복잡도

|연산|평균|최악 (충돌 많을 때)|
|---|---|---|
|삽입|**O(1)**|O(N)|
|삭제|**O(1)**|O(N)|
|탐색|**O(1)**|O(N)|

> 💡 평균적으로 O(1) — 배열 인덱스로 바로 접근하기 때문

---

## 충돌 (Collision)

> **서로 다른 Key가 같은 Index로 매핑되는 상황**

```
h("apple") = 3
h("grape") = 3  ← 같은 인덱스! → 충돌
```

### 충돌 해결 방법 1: 체이닝 (Chaining)

> 같은 인덱스에 **연결 리스트**로 여러 값을 연결

```
index 3 → ["apple":"사과"] → ["grape":"포도"] → null
index 4 → ["banana":"바나나"] → null
index 5 → null
```

```
장점: 구현 간단, 데이터 많아도 유연
단점: 포인터 추가 메모리 사용, 최악 O(N)
```

### 충돌 해결 방법 2: 개방 주소법 (Open Addressing)

> 충돌 시 **다른 빈 슬롯을 찾아서** 저장

#### 선형 탐사 (Linear Probing)

```
충돌 → 바로 다음 칸으로 이동

h("grape") = 3 → 3번 자리 사용 중
→ 4번 확인 → 비어있으면 저장
→ 4번도 사용 중이면 5번 확인 ...
```

#### 이중 해싱 (Double Hashing)

```
충돌 → 두 번째 해시 함수로 이동 간격 결정
→ 클러스터링 문제 완화
```

```
장점: 추가 메모리 없음
단점: 삭제 복잡, 테이블이 꽉 차면 성능 저하
```

---

## 로드 팩터 (Load Factor)

```
로드 팩터 = 저장된 데이터 수 / 테이블 크기

로드 팩터 높을수록 → 충돌 증가 → 성능 저하
일반적으로 0.7 이하 유지 권장
→ 초과 시 테이블 크기를 2배로 늘리고 재해싱
```

## C++ STL: unordered_map

> C++에서 해시 테이블 = `unordered_map`

```cpp
#include <unordered_map>
using namespace std;

unordered_map<string, int> um;

// 삽입
um["apple"] = 3;
um["banana"] = 5;
um.insert({"grape", 2});

// 탐색
cout << um["apple"];        // 3
cout << um.count("apple");  // 1 (있으면 1, 없으면 0)

// 삭제
um.erase("apple");

// 순회
for (auto& [key, val] : um) {
    cout << key << " : " << val << endl;
}

// 크기
cout << um.size();
```

### unordered_map 주요 함수

|함수|설명|
|---|---|
|`um[key] = val`|삽입 / 수정|
|`um.count(key)`|존재 여부 (0 또는 1)|
|`um.find(key)`|이터레이터 반환 (없으면 end())|
|`um.erase(key)`|삭제|
|`um.size()`|원소 개수|
|`um.empty()`|비어있으면 true|
|`um.clear()`|전체 삭제|

### find 안전하게 쓰기

```cpp
auto it = um.find("apple");
if (it != um.end()) {
    cout << it->second;  // 값 출력
} else {
    cout << "없음";
}
```

> ⚠️ `um["없는키"]` 하면 → 0으로 자동 초기화되며 삽입됨  
> 존재 여부 확인은 `count()` 또는 `find()` 사용!

## map vs unordered_map 비교

|구분|map|unordered_map|
|---|---|---|
|내부 구조|레드-블랙 트리|해시 테이블|
|탐색 속도|O(log N)|**O(1)** 평균|
|정렬|✅ Key 기준 정렬|❌ 순서 없음|
|최악 탐색|O(log N)|O(N)|
|사용 시기|정렬 필요할 때|빠른 탐색만 필요할 때|

---

## 해시 테이블 vs 다른 자료구조

|구분|배열|해시 테이블|BST(map)|
|---|---|---|---|
|탐색|O(N)|**O(1)**|O(log N)|
|삽입|O(1)|**O(1)**|O(log N)|
|정렬 유지|❌|❌|✅|
|메모리|적음|중간|중간|

## 자주 쓰는 패턴 (백준 기준)

### 빈도 수 세기

```cpp
unordered_map<int, int> freq;
for (int x : arr) {
    freq[x]++;          // 없으면 0으로 초기화 후 ++
}
```

### 중복 확인

```cpp
unordered_map<int, bool> visited;
if (!visited.count(x)) {
    visited[x] = true;  // 처음 방문
}
```

### 두 수의 합 (Two Sum)

```cpp
unordered_map<int, int> seen;
for (int i = 0; i < n; i++) {
    int need = target - arr[i];
    if (seen.count(need)) {
        // 정답: seen[need], i
    }
    seen[arr[i]] = i;
}
```

## 핵심 정리

```
해시 테이블:
  Key → 해시 함수 → Index → Value 저장
  평균 O(1) 탐색/삽입/삭제

충돌 해결:
  체이닝      → 연결 리스트로 연결
  개방 주소법 → 빈 칸 찾아서 저장

C++ STL:
  unordered_map → 해시 테이블
  map           → 레드-블랙 트리 (정렬 O)

주의:
  um["없는키"] → 자동 삽입됨!
  존재 확인은 count() 또는 find() 사용
```
