
# 🕸️ 비선형 자료구조 (Non-Linear Data Structures)

> [!abstract] **핵심 정의**
> 
> - **정의:** 데이터 요소들이 직선적으로 나열되지 않고, **1:N** 또는 **N:N**의 관계를 가지는 구조입니다.
>     
> - **특징:** 데이터가 계층적이거나 망 형태로 구성되어 있어, 하나의 데이터 뒤에 여러 개의 데이터가 올 수 있습니다.
>     

---

## 1. [Node] 선형 vs 비선형 자료구조 비교

| **구분**     | **선형 자료구조 (Linear)**   | **비선형 자료구조 (Non-Linear)**      |
| ---------- | ---------------------- | ------------------------------ |
| **데이터 관계** | **1:1** 관계 (앞뒤 하나씩 연결) | **1:N** 또는 **N:N** 관계          |
| **구성 형태**  | 직선, 연속적                | 비연속적, 계층적/망 형태                 |
| **순회 방식**  | 처음부터 끝까지 한 줄로 읽음       | **BFS(너비 우선)**, **DFS(깊이 우선)** |
| **난이도**    | 구현과 이해가 비교적 쉬움         | 구조가 복잡하고 알고리즘 설계가 어려움          |
| **종류**     | 배열, 연결 리스트, 스택, 큐      | **트리, 그래프, 힙, 트라이**            |

---

## 2. [Node] 대표적인 비선형 자료구조

### ① 트리 (Tree)

부모-자식 관계를 가지는 **계층적 구조**입니다.

- **구성:** 뿌리(Root), 가지(Branch), 잎(Leaf) 노드로 이루어집니다.
    
- **특징:** 사이클(Cycle)이 없는 그래프의 일종입니다.
    
- **용도:** 컴퓨터 파일 시스템, 도서 분류 체계, 대진표.
    

### ② 그래프 (Graph)

노드(Node)와 그 노드들을 잇는 간선(Edge)으로 이루어진 **망 구조**입니다.

- **구성:** 정점(Vertex)과 간선(Edge).
    
- **특징:** 방향성이 있을 수도 있고, 사이클이 존재할 수도 있습니다.
    
- **용도:** 네비게이션(최단 경로), SNS 인맥 지도, 인터넷 네트워크.

## 3. [Node] 실전 알고리즘에서의 활용 (C++)

비선형 자료구조는 주로 `std::map`, `std::set`, `std::priority_queue` 등을 통해 구현되거나, 직접 `struct`를 사용해 연결 관계를 정의합니다.

```cpp
#include <iostream>
#include <vector>

using namespace std;

// 간단한 트리 노드 구조 예시
struct node {
    int data;
    vector<node*> children; // 1:N 관계 (비선형)

    node(int val) : data(val) {}
};

int main() {
    // 입출력 최적화
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    // 루트 생성
    node* root = new node(1);
    
    // 자식 노드 추가 (비선형적 연결)
    root->children.push_back(new node(2));
    root->children.push_back(new node(3));
    root->children.push_back(new node(4));

    cout << "root data: " << root->data << "\n";
    cout << "number of children: " << root->children.size() << "\n";

    return 0;
}
```

## 🎓 논리적 설명 (중학생 수준)

비선형 자료구조는 '지도나 가계도'와 같습니다.

1. **선형 자료구조**는 '편의점 줄서기'예요. 내 앞사람이 누구인지, 뒷사람이 누구인지 딱 한 명씩만 정해져 있죠.
    
2. **비선형 자료구조**는 '지하철 노선도'와 같아요. 강남역에서 갈 수 있는 역이 한 곳인가요? 아니죠! 2호선을 타고 잠실로 갈 수도 있고, 신분당선을 타고 양재로 갈 수도 있어요.
    
3. 이렇게 한 곳에서 **여러 갈래로 길이 나뉘는 구조**를 비선형 자료구조라고 부릅니다. 더 복잡하지만, 세상의 많은 정보(인터넷, 도로, 족보 등)를 담기에는 이 구조가 훨씬 유리하답니다.
