# C++ 상속 (Inheritance)

---

## 상속이란?

> **기존 클래스의 멤버를 물려받아 새 클래스를 만드는 것**

```
부모 클래스 (Base / Super)    →   자식 클래스 (Derived / Sub)
    기존 기능 보유                   부모 기능 + 자기 기능 추가
```

```cpp
class Animal {          // 부모 클래스
    string name;
public:
    void eat() { cout << "먹는다" << endl; }
};

class Dog : public Animal {   // 자식 클래스 ← Animal 상속
    void bark() { cout << "멍멍" << endl; }
};

int main() {
    Dog d;
    d.eat();    // ✅ 부모 메서드 그대로 사용 가능
    d.bark();   // ✅ 자식 메서드
}
```

> 💡 **비유:** 부모님(Animal)의 재산(멤버)을 자식(Dog)이 물려받는 것  
> 자식은 부모 것도 쓰고, 자기 것도 추가 가능

---

## 상속 문법

```cpp
class 자식클래스 : 접근지정자 부모클래스 {
    // 추가 멤버
};
```

### 접근 지정자별 차이

|부모 멤버|public 상속|protected 상속|private 상속|
|---|---|---|---|
|public|public|protected|private|
|protected|protected|protected|private|
|private|접근 불가|접근 불가|접근 불가|

> 💡 **실무에서는 거의 항상 `public` 상속** 사용  
> private 멤버는 어떤 상속이든 자식이 직접 접근 불가

---

## 접근 지정자 3가지

```
public    → 어디서든 접근 가능
protected → 자기 클래스 + 자식 클래스만 접근 가능
private   → 자기 클래스만 접근 가능
```

```cpp
class Animal {
private:
    int age;        // 자식도 직접 접근 ❌
protected:
    string name;    // 자식은 접근 ✅, 외부는 ❌
public:
    void eat() { }  // 어디서든 ✅
};

class Dog : public Animal {
public:
    void info() {
        // age = 3;     ❌ private는 접근 불가
        name = "멍이";  // ✅ protected는 접근 가능
        eat();          // ✅ public은 접근 가능
    }
};
```

## 생성자/소멸자 호출 순서

```
생성: 부모 생성자 먼저 → 자식 생성자
소멸: 자식 소멸자 먼저 → 부모 소멸자  (생성의 역순)
```

```cpp
class Animal {
public:
    Animal()  { cout << "Animal 생성" << endl; }
    ~Animal() { cout << "Animal 소멸" << endl; }
};

class Dog : public Animal {
public:
    Dog()  { cout << "Dog 생성" << endl; }
    ~Dog() { cout << "Dog 소멸" << endl; }
};

int main() {
    Dog d;
}
// 출력:
// Animal 생성   ← 부모 먼저
// Dog 생성
// Dog 소멸      ← 자식 먼저
// Animal 소멸
```

## 부모 생성자 호출 (초기화 목록)

> 자식 생성자에서 부모 생성자에 인자를 넘길 때 사용

```cpp
class Animal {
    string name;
public:
    Animal(string n) : name(n) {
        cout << name << " 생성" << endl;
    }
};

class Dog : public Animal {
    string breed;
public:
    //                 ↓ 부모 생성자 호출
    Dog(string n, string b) : Animal(n), breed(b) { }
};

int main() {
    Dog d("멍이", "진돗개");
    // Animal("멍이") 먼저 호출 → Dog 생성자 실행
}
```

> ⚠️ 부모에 매개변수 있는 생성자만 있으면  
> 자식 초기화 목록에서 반드시 `Animal(n)` 처럼 명시해야 함

## 멤버 함수 재정의 (오버라이딩)

> **부모의 함수를 자식이 같은 이름으로 다시 정의**

```cpp
class Animal {
public:
    void sound() { cout << "..." << endl; }
};

class Dog : public Animal {
public:
    void sound() {          // 부모 함수를 재정의
        cout << "멍멍" << endl;
    }
};

int main() {
    Dog d;
    d.sound();              // "멍멍" ← 자식 버전 호출
    d.Animal::sound();      // "..."  ← 부모 버전 명시 호출
}
```

### 오버라이딩 vs 오버로딩 비교

|구분|오버라이딩|오버로딩|
|---|---|---|
|위치|부모-자식 간|같은 클래스 내|
|함수명|동일|동일|
|매개변수|동일|다름|
|목적|재정의|다양한 버전|

## 다형성과 virtual

> **포인터/참조로 부모 타입을 사용할 때 자식 버전 함수가 호출되게 하려면 `virtual` 필요**

```cpp
class Animal {
public:
    virtual void sound() { cout << "..." << endl; }
};

class Dog : public Animal {
public:
    void sound() override { cout << "멍멍" << endl; }
};

class Cat : public Animal {
public:
    void sound() override { cout << "야옹" << endl; }
};

int main() {
    Animal* a1 = new Dog();
    Animal* a2 = new Cat();

    a1->sound();    // "멍멍" ← virtual 덕분에 자식 버전 호출
    a2->sound();    // "야옹"
}
```

```
virtual 없으면 → 포인터 타입 기준 호출 (부모 버전)
virtual 있으면 → 실제 객체 타입 기준 호출 (자식 버전) ← 다형성
```

### override 키워드

```cpp
void sound() override { ... }   // 부모의 virtual 함수를 재정의함을 명시
                                // 오타 등 실수 시 컴파일 에러로 잡아줌
```

## 순수 가상 함수 & 추상 클래스

```cpp
class Animal {
public:
    virtual void sound() = 0;   // 순수 가상 함수 (= 0)
    //                    ↑ 구현 없음, 자식이 반드시 구현해야 함
};

// Animal a;    ❌ 추상 클래스는 객체 생성 불가
// Animal* a;   ✅ 포인터는 가능

class Dog : public Animal {
public:
    void sound() override { cout << "멍멍" << endl; }  // 반드시 구현
};
```

> 순수 가상 함수가 하나라도 있으면 → **추상 클래스**  
> 추상 클래스는 객체 직접 생성 불가, 포인터/참조만 가능

## 가상 소멸자

```cpp
class Animal {
public:
    virtual ~Animal() { cout << "Animal 소멸" << endl; }
    //↑ 반드시 virtual!
};

class Dog : public Animal {
public:
    ~Dog() { cout << "Dog 소멸" << endl; }
};

int main() {
    Animal* a = new Dog();
    delete a;
    // virtual 있으면: Dog 소멸 → Animal 소멸  ✅
    // virtual 없으면: Animal 소멸만           ❌ 메모리 누수!
}
```

> ⚠️ 부모 포인터로 자식 객체를 삭제할 때  
> 부모 소멸자에 `virtual` 없으면 자식 소멸자가 호출되지 않음 → **메모리 누수**

---

## 핵심 정리

```
상속 문법:    class 자식 : public 부모 { };
생성 순서:    부모 생성자 → 자식 생성자
소멸 순서:    자식 소멸자 → 부모 소멸자
부모 생성자 호출: 초기화 목록에서 부모클래스(인자)

virtual:
  - 부모 함수에 붙임 → 자식에서 오버라이딩 가능
  - 없으면 포인터 타입 기준 호출 (부모 버전)
  - 있으면 실제 객체 기준 호출 (자식 버전) = 다형성
  - 소멸자에도 반드시 virtual!

순수 가상 함수(= 0) → 추상 클래스 → 객체 생성 불가
```
