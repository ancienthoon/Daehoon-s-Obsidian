## 2.4 음수 표현

> 이전 방법(unsigned)은 양수만 표현 가능 → **MSB(맨 왼쪽 비트)를 부호 비트**로 사용: 0이면 양수, 1이면 음수

### 음수 표현 방법 3가지 (4비트에서 -4 표현 예)

|방법|-4 표현|설명|
|---|---|---|
|부호화-크기|**1**100|MSB=부호, 나머지=크기|
|1의 보수|**1**011|모든 비트 반전|
|2의 보수|**1**100|비트 반전 후 +1|

---

## 2.4.1 부호화-크기 표현

- MSB = 부호 비트 (0: +, 1: -)
- 나머지 (n-1)비트 = 수의 크기

**표현 범위:** -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1)

- 4비트: -7 ~ +7
- 5비트: -15 ~ +15

**예)**

```
+9  = 0 0001001
-9  = 1 0001001   ← MSB만 1로 바꿈

+35 = 0 0100011
-35 = 1 0100011
```

**2진수 → 10진수 변환 (간단한 방법)**

```
10100011 → MSB(1)는 부호(마이너스), 나머지 0100011 계산
→ -(32 + 2 + 1) = -35
```

**단점:**

1. 덧셈/뺄셈 시 부호 비트와 크기를 따로 처리해야 함 → 회로 복잡
2. 0이 두 개 (00000000 = +0, 10000000 = -0)

---

## 2.4.2 보수 표현

### 1의 보수

> **모든 비트를 반전 (0→1, 1→0)**

```
+9  = 0 0001001
-9  = 1 1110110  (1의 보수)
```

**문제점:** 0이 두 개 (00000000 = +0, 11111111 = -0) → 표현 범위: -127 ~ 127 (255가지)

---

### 2의 보수 ⭐ (실제 컴퓨터에서 사용)

> **모든 비트를 반전한 후 1을 더한다**

```
+9  = 0 0001001
-9  = 1 1110110  (비트 반전)
    +         1
    = 1 1110111  (2의 보수 = -9)
```

**핵심 성질:** 2의 보수를 취하면 부호만 바뀜

```
0101(+5) → 2의 보수 → 1011(-5)
1011(-5) → 2의 보수 → 0101(+5)
```

**0이 하나뿐:**

```
00000000 → 2의 보수 → (1)00000000 → 올림수 버림 → 00000000  ✓
```

→ 표현 범위: -128 ~ 127 (256가지) ← 1의 보수보다 1 더 많음

---

### 4비트 2의 보수 표현 요약

|2진수|10진수|
|---|---|
|0111|+7|
|0110|+6|
|0001|+1|
|0000|0|
|1111|-1|
|1110|-2|
|1000|**-8** (표현 가능한 최솟값)|

**n비트 2의 보수 표현 범위: -2ⁿ⁻¹ ~ +(2ⁿ⁻¹ - 1)**

|비트 수|표현 범위|
|---|---|
|4 bit|-8 ~ +7|
|8 bit|-128 ~ +127|
|16 bit|-32768 ~ +32767|
|32 bit|약 -21억 ~ +21억|

---

### 2진수 ↔ 10진수 변환 (2의 보수 방식)

#### ✅ 10진수 → 2진수

**양수:** 그냥 2진수로 변환

```
+5 → 0101
```

**음수:** 절댓값을 2진수로 바꾼 뒤 2의 보수 취함

```
-5 → 절댓값 5 = 0101 → 2의 보수 → 1011
```

#### ✅ 2진수 → 10진수

**MSB가 0 (양수):** 그냥 2진수 → 10진수

```
0101 → 5
```

**MSB가 1 (음수):** 2의 보수 취해서 크기 구한 뒤 마이너스 붙임

```
1011 → 2의 보수 → 0101 → 5 → -5
10101110 → 2의 보수 → 01010010 → 82 → -82
```

## 2.5 2의 보수 산술 연산

### 2의 보수 덧셈

> **그냥 더하고, MSB에서 올림수(carry) 발생하면 버린다**

**예)**

```
(a) (+4) + (+2) = +6
    0100
  + 0010
  ------
    0110  = +6  ✓

(b) (-5) + (+5) = 0
    1011  (-5의 2의 보수)
  + 0101
  ------
  (1)0000  → 올림수 버림 → 0000 = 0  ✓

(c) (-6) + (+2) = -4
    1010  (-6의 2의 보수)
  + 0010
  ------
    1100  = -4  ✓

(d) (-4) + (-1) = -5
    1100  (-4의 2의 보수)
  + 1111  (-1의 2의 보수)
  ------
  (1)1011  → 올림수 버림 → 1011 = -5  ✓
```

---

### 2의 보수 뺄셈

> **빼는 수(감수)의 2의 보수를 취해서 덧셈으로 바꾼다**

```
A - (+B) = A + (-B)   → +B의 2의 보수 취하고 더함
A - (-B) = A + (+B)   → -B의 2의 보수 취하고 더함
```

**예)**

```
(a) (+3) - (+7) = -4
    0011
  + 1001  (7의 2의 보수)
  ------
    1100  = -4  ✓

(b) (+5) - (+2) = +3
    0101
  + 1110  (2의 2의 보수)
  ------
  (1)0011  → 올림수 버림 → 0011 = +3  ✓
```

---

### 오버플로우(Overflow) ⚠️

> **결과값이 표현 범위를 초과 → 잘못된 결과 발생**

**오버플로우 발생 조건:**

- MSB끼리의 덧셈에서 나온 올림수 ≠ 그 바로 아래 비트 사이의 올림수

**예) 4비트 (-8 ~ +7 범위)**

```
(a) (+5) + (+6) = +11  → 범위 초과!
    0101
  + 0110
  ------
    1011  = -5  ← 잘못된 결과 (오버플로우)

(b) (-6) + (-7) = -13  → 범위 초과!
    1010
  + 1001
  ------
  (1)0011  = +3  ← 잘못된 결과 (오버플로우)
```

> 오버플로우 발생 시 MSB 쪽 carry와 그 아래 carry 값이 서로 다름
