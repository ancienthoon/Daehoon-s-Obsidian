## 왜 카노프 맵인가?

4장의 부울 대수 간략화는 **경험이 많이 필요**하고 **완전한 최소화가 어려움**

카노프 맵은 → **시각적으로** 1들을 묶어서 기계적으로 간략화 가능

|방법|특징|
|---|---|
|부울 대수|경험 필요, 최소화 보장 어려움|
|**카노프 맵**|시각적, 체계적, 최소화 보장|

> 변수 수가 5개 이상이면 → Quine-McClusky 방법 사용 (카노프 맵 한계)

---

## 5.1 3변수 카노프 맵

### 맵 구조

- 셀 수: 2³ = **8개**
- 행: A (0, 1)
- 열: BC (00, 01, 11, 10) ← **순서 주의! 그레이 코드 순서**

```
      BC
  A \ 00  01  11  10
   0 | 0 | 1 | 3 | 2 |
   1 | 4 | 5 | 7 | 6 |
```

> ⚠️ 열 순서가 00 → 01 → **11 → 10** (10진수 순서가 아님!) 이유: 인접한 셀끼리 딱 1비트만 달라야 하기 때문

### 셀 번호 ↔ 최소항 대응

```
      BC
  A \ 00   01   11   10
   0 | m₀ | m₁ | m₃ | m₂ |
   1 | m₄ | m₅ | m₇ | m₆ |
```

### 카노프 맵 채우는 법 (2가지)

#### 방법 1: 정규형 SOP로 변환 후 채우기 (비효율적)

1. 모든 항을 정규형으로 전개
2. 각 최소항 번호 → 해당 셀에 1 기입

#### 방법 2: 각 항이 1이 되는 조건 직접 찾기 ✅ (권장)

> **각 항에서 지정된 변수값 조건을 만족하는 셀 전부에 1 기입**

**예시: F(x,y,z) = x + y'z + x'yz'**

|항|조건|해당 셀|
|---|---|---|
|x|x = 1인 것 모두|x=1인 행 전체 (4, 5, 7, 6)|
|y'z|yz = 01인 것 모두|yz=01인 열 전체 (1, 5)|
|x'yz'|xyz = 010인 것|딱 그 셀 하나 (2)|

> 💡 **핵심:** 항에 없는 변수는 0이든 1이든 상관없음 → 해당되는 셀 전부 1

## 5.2 4변수 카노프 맵

### 맵 구조

- 셀 수: 2⁴ = **16개** (4×4)
- 행: AB (00, 01, 11, 10)
- 열: CD (00, 01, 11, 10)

```
      CD
 AB \ 00  01  11  10
  00 | 0 | 1 | 3 | 2 |
  01 | 4 | 5 | 7 | 6 |
  11 |12 |13 |15 |14 |
  10 | 8 | 9 |11 |10 |
```

> ⚠️ 행도 마찬가지: 00 → 01 → **11 → 10** 순서!

### 4변수 맵 채우는 법 (방법 2 적용)

**예시: F(A,B,C,D) = A'B + CD' + AB'C + BCD' + ABC'D'**

| 항      | 조건          | 셀 찾는 법            |
| ------ | ----------- | ----------------- |
| A'B    | AB = 01     | AB=01인 행 전체       |
| CD'    | CD = 10     | CD=10인 열 전체       |
| AB'C   | ABC = 101   | AB=10이고 C=1인 셀들   |
| BCD'   | BCD = 110   | B=1, C=1, D=0인 셀들 |
| ABC'D' | ABCD = 1100 | 딱 그 셀 하나          |

## 5.3 셀의 인접성을 이용한 간략화

### 인접성이란?

> **인접한 두 셀 = 딱 1비트만 다른 셀**

- 상하, 좌우로 붙어있는 셀
- 대각선 방향은 인접하지 않음
- **돌려감기(wrap-around):** 맵의 왼쪽 끝 ↔ 오른쪽 끝, 위쪽 끝 ↔ 아래쪽 끝도 인접!

```
인접한 셀들은 묶으면 → 다른 변수가 제거됨!

예: AB(11) + AB'(10) = A(B+B') = A
     ↑           ↑
   B=1인 셀   B=0인 셀  → B가 제거됨
```

---

### 묶음으로 간략화하는 3단계

```
① 인접한 1들을 묶는다 → 가능한 한 크게! (2ⁿ개 단위)
② 묶음 안에서 값이 바뀌는 변수 → 제거
③ 값이 변하지 않는 변수만 남김
   - 그 변수값이 0이면 → 보수형(A')으로
   - 그 변수값이 1이면 → 정상형(A)으로
```

### 간략화 예시 (3변수)

**F(A,B,C) = A'B'C + A'BC**

```
      BC
  A \ 00  01  11  10
   0 |   |   | 1 | 1 |   ← A=0행, BC=01과 11에 1
   1 |   |   |   |   |
```

- 두 1을 묶음
- A: 0으로 고정 → 변하지 않음 → A' 남김
- B: 0→1로 변함 → **제거**
- C: 1로 고정 → 변하지 않음 → C 남김
- 결과: **F = A'C**

---

### 묶음 크기별 규칙 ⭐

|묶음 크기|제거되는 변수 수|남는 변수 수 (3변수 기준)|
|---|---|---|
|1개 (2⁰)|0개|3개|
|2개 (2¹)|1개|2개|
|4개 (2²)|2개|1개|
|8개 (2³)|3개|0개 → F = 1|

> 💡 **묶음이 클수록 더 많이 간략화됨 → 최대한 크게 묶을 것!**

---

### 돌려감기 (Wrap-Around) 예시

```
      BC
  A \ 00  01  11  10
   0 | 1 |   |   | 1 |   ← 양 끝이 인접!
   1 | 1 |   |   | 1 |
```

- BC=00과 BC=10은 인접 (1비트만 다름: B만 다름)
- 위 아래도 묶임 → A'B'C' + AB'C' + A'BC' + ABC' = B'C'? 아니면...
- 4개 묶음 → A 제거, B 제거, C' 남음 → **F = C'**

---

### 핵심 패턴 암기

```
전체 열이 1 → 해당 열 변수만 남음
전체 행이 1 → 해당 행 변수만 남음
16개 전부 1 → F = 1
```

**자주 나오는 패턴 (4변수 기준):**

|패턴|결과|
|---|---|
|특정 열(CD) 전체 1|CD에 해당하는 항|
|특정 행(AB) 전체 1|AB에 해당하는 항|
|오른쪽 두 열(11, 10) 전체 1|C|
|왼쪽 두 열(00, 01) 전체 1|C'|
|위쪽 두 행(00, 01) 전체 1|A'|

---

### 간략화 절차 (실전용)

```
1단계: 맵 채우기
   → 각 항이 1이 되는 셀에 1 기입 (방법 2 사용)

2단계: 1 묶기
   → 처음에 겹치지 않게 크게 묶기 시작
   → 이후 남은 1들을 커버하기 위해 겹쳐서 추가 묶음 가능
   → 모든 1이 최소 하나의 묶음에 포함되어야 함

3단계: 각 묶음 → 항으로 변환
   → 변하지 않는 변수만 남기고 0이면 보수, 1이면 정상

4단계: 항들을 OR(+)로 연결
```

> ⚠️ **주의: 중복된 묶음 제거** 어떤 묶음의 모든 1이 다른 묶음에도 포함되어 있으면 → 그 묶음은 불필요!

---

### 오류 주의 예시

```
틀린 묶음:
- 3개 묶음 (2ⁿ개가 아님) → ❌
- 대각선 묶음 → ❌
- L자형 묶음 → ❌
- 이미 다른 묶음에 전부 포함된 묶음 → 중복, 제거 가능
```

### 맵 작성 예시 총정리

#### 예제 5-3 (3변수)

진리표에서 F가 1인 행들을 맵에 채우고 묶음

```
      yz
  x \ 00  01  11  10
   0 | 1 |   |   | 1 |
   1 | 1 | 1 |   | 1 |

묶음 1: 왼쪽 열(yz=00) 전체 + 오른쪽 열(yz=10) 전체 → z' 남음
묶음 2: x=1 행에서 yz=00, 01 → xy' 남음

F = xy' + z'
```

#### 예제 5-4 (4변수)

F(w,x,y,z) = Σ(0,1,2,4,5,6,8,9,12,13,14)

```
→ 0이 되는 셀: 3, 7, 10, 11, 15
→ 나머지 전부 1

묶음 1: y=0인 행들 전체 → y' (8개 묶음)
묶음 2: 왼쪽 두 열(z=0) 중 w=0행 → w'z'
묶음 3: 오른쪽 두 열(x=1, z=0) → xz'

F = y' + w'z' + xz'
```

## 묶음 규칙 총정리

### 반드시 지킬 규칙

```
✅ 묶음 크기는 반드시 1, 2, 4, 8, 16 (2ⁿ개)
✅ 묶음은 직사각형(정사각형 포함) 모양만 가능
✅ 가능한 한 크게 묶기
✅ 모든 1은 반드시 하나 이상의 묶음에 포함
✅ 같은 1을 여러 묶음에서 중복 사용 가능
✅ 돌려감기 가능 (양 끝 인접)

❌ 대각선 묶음 불가
❌ L자, T자 모양 불가
❌ 3개, 5개, 6개 등 2ⁿ이 아닌 크기 불가
❌ 불필요한 중복 묶음 (다른 묶음에 이미 전부 포함된 것)
```

### 묶음 → 항 변환 규칙

```
묶음 내에서:
- 값이 항상 0인 변수 → 보수형 (A')
- 값이 항상 1인 변수 → 정상형 (A)
- 값이 0도 있고 1도 있는 변수 → 제거 (해당 변수 사라짐)
```

### 간략화 결과 크기 직관

|남는 변수 수|묶음 크기 (3변수 맵)|묶음 크기 (4변수 맵)|
|---|---|---|
|변수 3개|1개|-|
|변수 2개|2개|1개|
|변수 1개|4개|2개|
|변수 0개 (F=1)|8개|4개|

> 💡 **n변수 맵에서 2ᵏ개 묶음 → (n-k)개 변수 남음**

