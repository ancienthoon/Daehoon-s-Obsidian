## 1. 투 포인터 기초 (Two Pointers Basics)

> [!abstract] **핵심 정의**
> 
> - **개념:** 리스트나 배열에서 **두 개의 지점(인덱스)**을 조절해가며 원하는 값을 찾는 알고리즘입니다.
>     
> - **왜 쓰는가?** 보통 중첩 반복문($O(n^2)$)으로 풀어야 하는 문제를 선형 시간(**$O(n)$**) 안에 해결하기 위해 사용합니다.
>     
> - **조건:** 대부분의 경우 배열이 **정렬**되어 있거나, **연속된 구간**을 다룰 때 성능이 극대화됩니다.
>     

---

## 2. 두 포인터의 이동 방식

투 포인터는 크게 두 가지 유형으로 나뉩니다.

### ① 같은 방향에서 시작 (Start, End)

- **방식:** 두 포인터가 모두 인덱스 0에서 시작하여 오른쪽으로 이동합니다.
    
- **주요 용도:** "연속된 부분 배열의 합이 $k$인 구간 찾기" 등.
    
- **동작:** * 합이 $k$보다 작으면 `end`를 오른쪽으로.
    
    - 합이 $k$보다 크거나 같으면 `start`를 오른쪽으로.
        

### ② 양끝에서 시작 (Left, Right)

- **방식:** 하나는 0, 하나는 $n-1$ 인덱스에서 시작하여 가운데로 모입니다.
    
- **주요 용도:** **정렬된 배열**에서 "두 수의 합이 $k$가 되는 쌍 찾기", "회문(Palindrome) 판별".
    
- **동작:**
    
    - 두 수의 합이 $k$보다 작으면 `left++` (값을 키움).
        
    - 두 수의 합이 $k$보다 크면 `right--` (값을 줄임).
        

---

## 3. [Node] 실전 코드: 특정한 합을 가지는 부분 연속 수열

백준에서 아주 흔하게 볼 수 있는 유형입니다.

```cpp
#include <iostream>
#include <vector>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    int n = 5; // 데이터 개수
    int m = 5; // 찾고자 하는 부분합
    int arr[] = {1, 2, 3, 2, 5};

    int count = 0;
    int interval_sum = 0;
    int end = 0;

    // start를 차례대로 이동시키며 반복
    for (int start = 0; start < n; start++) {
        // end를 가능한 만큼 이동시키기
        while (interval_sum < m && end < n) {
            interval_sum += arr[end];
            end++;
        }
        // 부분합이 m일 때 카운트 증가
        if (interval_sum == m) {
            count++;
        }
        // 다음 루프로 가기 전 start가 가리키던 값 빼기
        interval_sum -= arr[start];
    }

    cout << count << "\n";

    return 0;
}
```

## 4. 투 포인터 vs 슬라이딩 윈도우

두 기법은 비슷해 보이지만 결정적인 차이가 있습니다.

|**구분**|**투 포인터 (Two Pointers)**|**슬라이딩 윈도우 (Sliding Window)**|
|---|---|---|
|**구간 길이**|**가변적** (포인터 간격이 계속 변함)|**고정적** (창문의 크기가 일정함)|
|**포인터**|두 개의 포인터가 독립적으로 움직임|구간 자체가 통째로 옆으로 밀려남|
|**비유**|자를 늘렸다 줄였다 하며 측정|정해진 크기의 돋보기를 옆으로 밀며 관찰|

## 🎓 백준 골드 공략 팁

> [!tip] **언제 투 포인터를 떠올려야 할까?**
> 
> 1. 입력값 $n$이 **100,000 이상**이라서 $O(n^2)$으로는 절대 못 풀 때.
>     
> 2. "연속된", "구간", "두 수의 쌍"이라는 키워드가 보일 때.
>     
> 3. 만약 정렬이 안 되어 있다면, **직접 정렬($O(n \log n)$)**을 한 뒤 투 포인터를 써도 되는지 먼저 고민해 보세요.


# 🔄 순환 알고리즘 (Recursion)

> [!abstract] **핵심 정의**
> 
> - **자기 호출:** 함수 내부에서 자기 자신을 다시 호출하는 방식입니다.
>     
> - **분할 정복:** 크고 복잡한 문제를 동일한 구조의 작은 문제들로 나누어 해결합니다.
>     
> - **비유:** 러시아 인형 '마트료시카'처럼, 인형을 열면 똑같이 생긴 더 작은 인형이 계속 나오는 것과 같습니다.
>     

---

## 1. 순환의 2대 필수 조건 (Golden Rules)

순환 함수가 무한 루프에 빠지지 않으려면 반드시 다음 두 가지가 포함되어야 합니다.

1. **기본 케이스 (Base Case):** 더 이상 순환하지 않고 종료되는 조건입니다. (멈춤 버튼)
    
2. **순환 케이스 (Recursive Case):** 문제를 더 작게 만들어 자기 자신을 호출하는 부분입니다. (계속 진행)
    

> [!warning] **스택 오버플로 (Stack Overflow)**
> 
> 기본 케이스가 없거나 잘못 설정되면, 함수 호출 정보가 메모리의 '스택' 영역에 무한히 쌓이다가 결국 프로그램이 강제 종료됩니다.

---

## 2. 순환 vs 반복 (Recursion vs Iteration)

|**구분**|**순환 (Recursion)**|**반복 (Iteration)**|
|---|---|---|
|**구현 방식**|함수 자기 자신 호출|`for`, `while` 문 사용|
|**코드 가독성**|알고리즘의 정의를 그대로 옮겨와 간결함|코드가 길어질 수 있으나 직관적임|
|**메모리 사용**|**스택 메모리**를 많이 소모함|메모리 소모가 상대적으로 적음|
|**속도**|함수 호출 오버헤드로 인해 약간 느림|일반적으로 더 빠름|

---

## 3. 대표적인 사례: 팩토리얼 (Factorial)

팩토리얼 $n!$은 $n \times (n-1)!$로 정의됩니다.

### 💻 C++ 구현 예시

```cpp
#include <iostream>

using namespace std;

int factorial(int n) {
    // 1. 기본 케이스
    if (n <= 1) return 1;
    
    // 2. 순환 케이스
    return n * factorial(n - 1);
}

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    int num = 5;
    cout << num << "! = " << factorial(num) << "\n";

    return 0;
}
```

## 4. 순환의 효율성 문제와 해결책

순환은 코드가 우아하지만, **피보나치 수열**처럼 동일한 계산을 반복하는 경우 심각한 성능 저하가 발생합니다.

- **문제점:** $f(5)$를 구하기 위해 $f(3)$을 여러 번 중복 계산함 ($O(2^n)$의 시간 복잡도).
    
- **해결책:** 1. **메모이전(Memoization):** 이미 계산한 값을 배열에 저장해두고 재사용합니다.
    
    2. **꼬리 순환(Tail Recursion):** 순환 호출이 함수의 마지막 연산이 되도록 하여 컴파일러가 최적화할 수 있게 만듭니다.
