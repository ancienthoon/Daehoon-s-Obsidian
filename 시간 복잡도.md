# 1. 알고리즘의 성능 분석 기법

> [!abstract] **성능 분석의 두 가지 관점**
> 
> 알고리즘의 우수성을 판단하기 위해 '얼마나 빠른가'를 측정하는 방법은 크게 두 가지로 나뉩니다.

### ① 수행 시간 측정 (Experimental Analysis)

- **방법:** 두 알고리즘을 실제로 프로그래밍하여 실행 시간을 초 단위로 측정.
    
- **한계:** * 반드시 구현이 필요함.
    
    - 동일한 하드웨어 환경에서 테스트해야 공정함.
        
    - 데이터의 양이나 하드웨어 성능에 따라 결과가 왜곡될 수 있음.
        

### ② 알고리즘의 복잡도 분석 (Complexity Analysis)

- **방법:** 구현하지 않고도 이론적으로 수행 시간을 분석.
    
- **핵심:** 알고리즘이 수행하는 **연산의 횟수**를 측정.
    
- **특징:** 연산 횟수는 대개 입력 자료의 개수($n$)에 대한 함수로 나타냄.

# 2. 시간 복잡도와 빅오 표기법 (Big-O)

> [!info] **점근적 표기법 (Asymptotic Notation)**
> 
> 자료의 개수 $n$이 무한히 커질 때, 수행 시간이 어떻게 증가하는지 그 **'상한(Upper Bound)'**을 표시하는 방법입니다.

### 빅오 표기법의 정의

- 모든 $n \ge n_0$에 대하여 $|f(n)| \le c|g(n)|$을 만족하는 상수 $c$와 $n_0$가 존재하면 $f(n) = O(g(n))$이다.
    
- 즉, "아무리 최악의 상황이라도 $g(n)$의 속도보다는 빠르다"는 것을 보장합니다.
    

### 증가 속도에 따른 효율성 비교

오른쪽으로 갈수록 데이터가 늘어날 때 수행 시간이 기하급수적으로 늘어나는 비효율적인 알고리즘입니다.

1. $O(1)$ : 상수 시간 (입력값에 상관없이 일정)
    
2. $O(\log n)$ : 로그 시간
    
3. $O(n)$ : 선형 시간
    
4. $O(n \log n)$ : 선형 로그 시간
    
5. $O(n^2)$ : 제곱 시간
    
6. $O(2^n)$ : 지수 시간
    
7. $O(n!)$ : 팩토리얼 시간

# 3. 프로그램 효율성의 중요성 (사례 분석)

> [!example] **$n^2$ vs $2^n$ 알고리즘 비교**
> 
> 입력 데이터의 개수($n$)가 작을 때는 차이가 미미하지만, 커질수록 천문학적인 차이가 발생합니다.

|**입력 개수 (n)**|**프로그램 A (n2)**|**프로그램 B (2n)**|**비고**|
|---|---|---|---|
|**$n=6$**|36초|64초|큰 차이 없음|
|**$n=100$**|**10,000초** (약 2.7시간)|**$4 \times 10^{22}$년**|**수행 불가능**|

- **결론:** 하드웨어 성능을 높이는 것보다, 알고리즘의 복잡도(Big-O)를 낮추는 것이 대용량 데이터 처리에 훨씬 결정적인 영향을 미칩니다.

# 4. 빅오 이외의 표기법 (Omega & Theta)

> [!tip] **최선의 경우와 평균의 경우**
> 
> 알고리즘 분석 시 상한선(Big-O) 외에도 하한선과 평균치를 고려할 수 있습니다.

### ① 빅오메가 표기법 ($\Omega$)

- **의미:** 함수의 **하한(Lower Bound)**을 표시.
    
- **용도:** 최선의 경우(Best Case) 분석.
    
- **정의:** 모든 $n \ge n_0$에 대하여 $|f(n)| \ge c|g(n)|$을 만족하는 $c, n_0$ 존재 시 $f(n) = \Omega(g(n))$.
    

### ② 빅세타 표기법 ($\Theta$)

- **의미:** 상한인 동시에 하한을 표시.
    
- **용도:** **평균적인 경우(Average Case)** 분석. 가장 정확한 성능 지표.
    
- **정의:** $f(n) = O(g(n))$이면서 $f(n) = \Omega(g(n))$이면 $f(n) = \Theta(g(n))$.
