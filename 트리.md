
# 🌲 트리 (Tree) 자료구조

> [!abstract] **핵심 정의**
> 
> - **비선형 구조:** 데이터가 일렬로 나열되지 않고 부모-자식 관계를 가지는 **계층적 구조**입니다.
>     
> - **사이클 없음:** 그래프의 일종이지만, 회로(Cycle)가 존재하지 않는 연결 그래프입니다.
>     
> - **특징:** 하나의 루트 노드에서 시작하여 여러 개의 자식 노드로 뻗어나갑니다.
>     

---

## 1. 주요 용어 정리

트리 구조를 이해하기 위해 반드시 알아야 할 용어들입니다.

- **루트(Root):** 트리의 최상단 노드 (가장 꼭대기).
    
- **노드(Node):** 트리를 구성하는 각 데이터 요소.
    
- **에지(Edge):** 노드와 노드를 잇는 연결선 (가지).
    
- **리프(Leaf):** 자식이 없는 최하단 노드 (단말 노드).
    
- **부모/자식(Parent/Child):** 연결된 노드 중 상위 노드와 하위 노드.
    
- **깊이(Depth):** 루트에서 특정 노드까지 도달하기 위해 거치는 간선의 수.
    
- **높이(Height):** 리프 노드에서 루트 노드까지의 가장 긴 경로의 수.
    

---

## 2. 트리의 종류 요약표

| **종류**                  | **특징**                        | **주요 용도**                    |
| ----------------------- | ----------------------------- | ---------------------------- |
| **이진 트리 (Binary Tree)** | 각 노드가 최대 2개의 자식만 가짐           | 가장 기본적인 트리 형태                |
| **이진 탐색 트리 (BST)**      | 왼쪽 자식 < 부모 < 오른쪽 자식 규칙 적용     | 데이터 검색, 정렬                   |
| **AVL 트리**              | 좌우 서브트리의 높이 차를 1 이하로 유지       | 균형 잡힌 빠른 검색                  |
| **레드-블랙 트리**            | 노드에 색상을 부여하여 균형을 유지           | C++ `std::set`, `map`의 내부 구현 |
| **힙 (Heap)**            | 최댓값이나 최솟값을 빠르게 찾기 위한 완전 이진 트리 | 우선순위 큐 구현                    |
| **B-트리**                | 하나의 노드에 여러 데이터와 자식을 가짐        | 데이터베이스, 파일 시스템 인덱스           |

---

## 3. 이진 탐색 트리(BST)의 성질

알고리즘 문제 풀이에서 가장 자주 등장하는 형태입니다.

1. **효율적 탐색:** 평균적으로 $O(\log N)$의 시간 복잡도로 데이터를 찾을 수 있습니다.
    
2. **중위 순회(In-order):** 이진 탐색 트리를 중위 순회하면 데이터가 **오름차순**으로 정렬되어 나옵니다.
    
3. **한계:** 데이터가 한쪽으로 치우쳐서 입력될 경우(예: 1, 2, 3, 4 순서) 일직선 형태가 되어 탐색 속도가 $O(N)$으로 느려집니다. (이를 해결한 것이 AVL이나 레드-블랙 트리입니다.)

## 💻 C++ 실전 코드: 노드 구조 및 생성

가장 기본적인 이진 트리의 노드 구조체 예시입니다.

```cpp
#include <iostream>

using namespace std;

// 노드 구조 정의
struct node {
    int data;
    node* left;
    node* right;

    node(int val) {
        data = val;
        left = nullptr;
        right = nullptr;
    }
};

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    // 간단한 트리 생성
    //       1
    //      / \
    //     2   3
    node* root = new node(1);
    root->left = new node(2);
    root->right = new node(3);

    cout << "루트 노드 데이터: " << root->data << "\n";
    cout << "왼쪽 자식 데이터: " << root->left->data << "\n";

    return 0;
}
```

## 🎓 학습 포인트: 면접 및 시험 대비

- **(질문) "트리와 그래프의 가장 큰 차이점은?"**
    
    - **(답변)** "트리는 방향성이 있고 사이클(회로)이 없는 특수한 형태의 그래프입니다. 또한 모든 자식 노드는 단 하나의 부모 노드만을 가져야 한다는 계층적 제약이 있습니다."
        
- **(질문) "편향 트리(Skewed Tree)란 무엇인가요?"**
    
    - **(답변)** "모든 노드가 왼쪽 혹은 오른쪽 한 방향으로만 자식을 가져서 마치 연결 리스트처럼 일렬로 나열된 트리를 말합니다. 이 경우 이진 탐색 트리의 장점인 빠른 탐색 성능을 잃게 됩니다."
