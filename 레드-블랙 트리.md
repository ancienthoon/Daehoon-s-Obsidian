
## 레드-블랙 트리 (Red-Black Tree) 🔴⚫

---

### 1. 왜 레드-블랙 트리가 필요할까요?

2강에서 배운 BST의 치명적 약점 기억하나요?

```
삽입 순서가 [1, 2, 3, 4, 5] 이면?

[1]
  \
  [2]
    \
    [3]
      \
      [4]
        \
        [5]

트리가 완전히 한쪽으로 기울어짐 💀
탐색이 O(log N) → O(N) 으로 망가짐
```

> 💡 **해결책 = 트리가 기울어질 때마다 자동으로 균형을 맞춰주자!** 이걸 구현한 게 **균형 이진 탐색 트리** 이고, 그 중 가장 널리 쓰이는 게 **레드-블랙 트리**예요. (C++ STL의 `map`, `set`이 내부적으로 이걸 사용해요!)

---

### 2. 핵심 규칙 — 5가지

레드-블랙 트리는 BST 규칙에 **색깔 규칙**을 추가해요.

```
① 모든 노드는 빨강(🔴) 또는 검정(⚫) 이다.
② 루트 노드는 항상 검정(⚫) 이다.
③ 모든 리프(NULL 노드)는 검정(⚫) 이다.
④ 빨강 노드의 자식은 반드시 검정 이다.
   (빨강이 연속으로 나올 수 없음 🚫)
⑤ 어떤 노드에서 리프까지의 경로에서
   검정 노드의 수는 모두 같다. (Black-Height)
```

> 💡 **이 규칙들이 왜 균형을 보장하나요?** 규칙 ④⑤ 때문에 트리의 가장 긴 경로는 가장 짧은 경로의 **최대 2배**를 넘을 수 없어요. → 트리 높이가 항상 **O(log N)** 으로 유지돼요!

---

### 3. 시각적 구조

```
올바른 레드-블랙 트리 예시:

           ⚫30
          /    \
       🔴20    🔴40
       /  \    /  \
     ⚫10 ⚫25 ⚫35 ⚫50

규칙 확인:
① 모든 노드 빨강/검정 ✅
② 루트(30) 검정 ✅
④ 빨강(20)의 자식(10, 25) 모두 검정 ✅
   빨강(40)의 자식(35, 50) 모두 검정 ✅
⑤ 루트→리프 경로의 검정 노드 수:
   30→20→10 : 검정 2개
   30→20→25 : 검정 2개
   30→40→35 : 검정 2개
   30→40→50 : 검정 2개  모두 같음 ✅
```

---

### 4. 균형을 맞추는 방법 — 회전과 재색칠

삽입/삭제 후 규칙이 깨지면 두 가지 방법으로 고쳐요.

#### 🔄 회전 (Rotation)

```
왼쪽 회전 (Left Rotation) — x를 기준으로:

      x                  y
     / \      →         / \
    A   y              x   C
       / \            / \
      B   C          A   B

오른쪽 회전 (Right Rotation) — y를 기준으로:

      y                  x
     / \      →         / \
    x   C              A   y
   / \                    / \
  A   B                  B   C
```

#### 🎨 재색칠 (Recoloring)

```
삽입 후 빨강-빨강 충돌 발생 시:

삼촌 노드가 빨강이면 → 재색칠로 해결
      ⚫G                  🔴G
      / \       →          / \
   🔴P  🔴U             ⚫P  ⚫U
   /                    /
 🔴X                  🔴X

삼촌 노드가 검정이면 → 회전으로 해결
```

---

### 5. 삽입 과정 예시

```
[10, 20, 30] 순서로 삽입:

Step 1: 10 삽입
  🔴10  →  ⚫10  (루트는 항상 검정)

Step 2: 20 삽입 (오른쪽 자식, 빨강으로)
  ⚫10
     \
     🔴20  ← 규칙 위반 없음 ✅

Step 3: 30 삽입 (빨강으로)
  ⚫10
     \
     🔴20
        \
        🔴30  ← 🚫 빨강-빨강 연속! 규칙 ④ 위반!

  → 왼쪽 회전 + 재색칠:

     ⚫20
     /   \
  🔴10   🔴30  ✅ 균형 복구!
```

---

### 6. BST / AVL 트리 / 레드-블랙 트리 비교

||**BST**|**AVL 트리**|**레드-블랙 트리**|
|---|---|---|---|
|균형 보장|❌|✅ 엄격|✅ 유연|
|탐색 속도|O(N) 최악|O(log N)|O(log N)|
|삽입 속도|O(N) 최악|O(log N)|O(log N)|
|균형 기준|없음|높이 차 ≤ 1|색깔 규칙|
|회전 횟수|없음|많음|적음|
|실전 사용|교육용|DB 인덱스|**STL map/set**|

> 💡 **AVL vs 레드-블랙**
> 
> - AVL은 더 엄격하게 균형을 맞춰서 **탐색이 약간 더 빠름**
> - 레드-블랙은 균형 조건이 유연해서 **삽입/삭제가 더 빠름**
> - 삽입/삭제가 잦은 실전에서는 **레드-블랙이 더 유리**해서 STL이 레드-블랙을 채택했어요!

---

### 7. 시간 복잡도

|연산|시간복잡도|이유|
|---|---|---|
|탐색|**O(log N)**|트리 높이 항상 O(log N) 보장|
|삽입|**O(log N)**|삽입 후 회전/재색칠 최대 O(log N)|
|삭제|**O(log N)**|삭제 후 회전/재색칠 최대 O(log N)|

---

### ✅ 핵심 정리

```
레드-블랙 트리 = BST + 색깔 규칙 5가지
목적           = 트리 높이를 항상 O(log N)으로 유지 (균형 보장)
균형 방법      = 회전(Rotation) + 재색칠(Recoloring)
실전 활용      = C++ STL map, set, multimap, multiset 내부 구현체

직접 구현은 복잡해서 실전/코딩테스트에서는
그냥 map/set을 쓰면 돼요! 😊
```

# 📦 C++ STL: set (집합)

> [!abstract] **핵심 정의**
> 
> - **중복 불허:** 모든 원소는 유일해야 합니다.
>     
> - **자동 정렬:** 원소를 넣자마자 내부적으로 오름차순 정렬됩니다.
>     
> - **자료구조:** 주로 **Red-Black Tree** (균형 이진 탐색 트리)로 구현되어 있습니다.
>     

---

## 1. 주요 특징 및 성능

|**특징**|**내용**|**비유**|
|---|---|---|
|**순서**|입력 순서와 상관없이 **값의 크기 순**으로 저장됨|사전식 배열|
|**중복**|이미 존재하는 값을 넣으면 무시됨|출석부의 고유 번호|
|**속도**|탐색, 삽입, 삭제 모두 **$O(\log N)$**|이진 탐색의 효율성|

---

## 2. 핵심 멤버 함수 (Usage)

| **함수**        | **기능**        | **비고**                              |
| ------------- | ------------- | ----------------------------------- |
| `s.insert(k)` | 원소 `k`를 삽입    | 삽입 후 자동 정렬됨                         |
| `s.erase(k)`  | 원소 `k`를 삭제    | 값이 없으면 아무 일도 안 함                    |
| `s.find(k)`   | 원소 `k`를 탐색    | 찾으면 반복자(iterator), 없으면 `s.end()` 반환 |
| `s.count(k)`  | 원소 `k`의 개수 확인 | 존재하면 1, 없으면 0 (중복이 없으므로)            |
| `s.clear()`   | 모든 원소 삭제      | 메모리는 유지될 수 있음                       |

---

## 💻 실전 코드 예시

사용자께서 선호하시는 입출력 최적화 및 `using namespace std;`를 적용한 예제입니다.

```cpp
#include <iostream>
#include <set>

using namespace std;

int main() {
    // 입출력 최적화
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    set<int> s;

    // 1. 데이터 삽입 (순서 뒤죽박죽)
    s.insert(7);
    s.insert(3);
    s.insert(9);
    s.insert(3); // 중복 삽입 시도 (무시됨)

    // 2. 전체 원소 출력 (자동으로 3 7 9 출력됨)
    cout << "set 원소: ";
    for (int x : s) {
        cout << x << " ";
    }
    cout << "\n";

    // 3. 원소 존재 여부 확인 (find 활용)
    if (s.find(7) != s.end()) {
        cout << "7이 존재합니다.\n";
    }

    // 4. 원소 삭제
    s.erase(7);
    cout << "7 삭제 후 개수: " << s.size() << "\n";

    return 0;
}
```

## 🎓 편입 면접 대비: 핵심 질문

> [!question] **`set`과 `unordered_set`의 결정적인 차이는 무엇인가요?**
> 
> **Answer:** "가장 큰 차이는 **정렬 여부**와 **시간 복잡도**입니다. `set`은 이진 트리를 사용하여 $O(\log N)$의 속도로 데이터를 정렬된 상태로 유지합니다. 반면 `unordered_set`은 해시 테이블을 사용하여 정렬은 하지 않지만, 평균 **$O(1)$**의 매우 빠른 탐색 속도를 가집니다."

> [!question] **`set`에 내가 만든 클래스(사용자 정의 객체)를 넣으려면 무엇이 필요한가요?**
> 
> **Answer:** "`set`은 원소를 저장할 때 크기를 비교하여 정렬해야 하므로, 해당 클래스 내부에 **`<` (less than) 연산자 오버로딩**이 반드시 정의되어 있어야 합니다. 그렇지 않으면 컴파일 에러가 발생합니다."


> [!question] **`map`에서 `[]` 연산자를 사용할 때 주의할 점은 무엇인가요?**
> 
> **Answer:** "`[]` 연산자는 해당 키가 존재하지 않을 경우, **자동으로 기본값을 가진 새로운 요소를 생성**해버립니다. 따라서 단순히 존재 여부만 확인하고 싶을 때는 `find()`나 `count()` 함수를 사용하는 것이 메모리 관리 측면에서 더 안전합니다."

> [!question] **`map`의 시간 복잡도가 $O(\log N)$인 이유는 무엇인가요?**
> 
> **Answer:** "내부적으로 균형 이진 탐색 트리인 **레드-블랙 트리**를 사용하기 때문입니다. 데이터를 삽입하거나 찾을 때마다 트리의 높이만큼만 비교하면 되므로, 데이터가 $N$개일 때 약 $\log_2 N$번의 비교만으로 작업을 수행할 수 있습니다."

# 🗺️ C++ STL: map (연관 컨테이너)

> [!abstract] **핵심 정의**
> 
> - **Key-Value 쌍:** 데이터를 '이름(Key)'과 '실제 값(Value)'의 쌍으로 묶어서 저장합니다.
>     
> - **자동 정렬:** 원소를 넣으면 **Key를 기준으로 자동 오름차순 정렬**됩니다.
>     
> - **중복 불허:** 하나의 Key는 단 하나의 Value만 가질 수 있습니다. (Key는 유일해야 함)
>     
> - **자료구조:** 내부적으로 **레드-블랙 트리(Red-Black Tree)**로 구현되어 있습니다.
>     

---

## 1. `map` vs `unordered_map`

|**특징**|**std::map**|**std::unordered_map**|
|---|---|---|
|**내부 구조**|레드-블랙 트리 (이진 트리)|해시 테이블|
|**정렬 여부**|**Key 기준 자동 정렬**|정렬되지 않음|
|**시간 복잡도**|$O(\log N)$|**평균 $O(1)$**|
|**사용 시점**|데이터가 정렬되어야 할 때|정렬 필요 없이 빠른 탐색이 필요할 때|

---

## 2. 주요 멤버 함수

|**함수**|**기능**|**비고**|
|---|---|---|
|`m[key] = val`|데이터 삽입 및 수정|가장 직관적인 방법|
|`m.insert({k, v})`|데이터 삽입|이미 키가 있으면 삽입 실패|
|`m.erase(key)`|특정 키를 가진 요소 삭제||
|`m.find(key)`|키를 찾아 반복자 반환|못 찾으면 `m.end()` 반환|
|`m.count(key)`|키의 존재 여부 확인|있으면 1, 없으면 0 반환|
|`m.clear()`|모든 요소 삭제||

---

## 💻 실전 코드 예시

사용자 정의 규칙(소문자 변수, 최소 주석, 입출력 최적화)을 적용한 예제입니다.

```cpp
#include <iostream>
#include <map>
#include <string>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    // map 선언 <Key타입, Value타입>
    map<string, int> score;

    // 1. 데이터 삽입
    score["kim"] = 90;
    score["lee"] = 85;
    score["park"] = 95;

    // 2. 데이터 수정
    score["kim"] = 100;

    // 3. 전체 순회 (자동 정렬되어 출력됨)
    // first는 Key, second는 Value를 의미
    for (auto const& pair : score) {
        cout << pair.first << " : " << pair.second << "\n";
    }

    // 4. 특정 데이터 찾기
    string name = "lee";
    if (score.find(name) != score.end()) {
        cout << name << "의 점수는 " << score[name] << "입니다.\n";
    }

    return 0;
}
```

# ⚖️ std::set vs std::map 비교

> [!abstract] **핵심 요약**
> 
> - **set:** "이 값이 명단에 있는가?" (**존재 여부**와 **정렬**이 중요할 때)
>     
> - **map:** "이 이름의 점수는 몇 점인가?" (**연관 데이터**를 찾을 때)
>     

---

## 1. 주요 차이점 요약표

|**구분**|**std::set**|**std::map**|
|---|---|---|
|**데이터 구성**|**Key** (원소 그 자체가 키)|**Key + Value** (쌍으로 저장)|
|**저장 형태**|`value`|`pair<const Key, Value>`|
|**접근 방식**|반복자(iterator), find()|**`[]` 연산자**, find(), at()|
|**주요 용도**|중복 제거, 특정 값의 존재 확인|사전(Dictionary), ID별 정보 저장|
|**공통점**|**중복 불가**, **자동 정렬**, $O(\log N)$ 성능|**중복 불가**, **자동 정렬**, $O(\log N)$ 성능|

---

## 2. 결정적인 차이 세 가지

### ① 데이터의 구조

- **set:** 집합(Set) 개념입니다. 원소 자체가 곧 키(Key)이기 때문에, `set<int> s;`에 `10`을 넣으면 그냥 `10`이 저장됩니다.
    
- **map:** 지도(Map) 또는 사전 개념입니다. `map<string, int> m;`에 `{"apple", 1000}`을 넣으면 `apple`이라는 키를 통해 `1000`이라는 값에 접근합니다.
    

### ② 데이터 접근 (Access)

- **set:** 특정 원소에 바로 접근하는 인덱스 연산자가 없습니다. `find()`를 통해 반복자를 얻어야 합니다.
    
- **map:** `m["apple"]`과 같이 **대괄호(`[]`) 연산자**를 사용하여 키에 대응하는 값에 매우 직관적으로 접근할 수 있습니다.
    

### ③ 활용 시나리오

- **set:** "오늘 출석한 학생들의 번호 목록(중복 제거 및 정렬)"을 관리할 때 유용합니다.
    
- **map:** "학생 이름(Key)별 수학 점수(Value)"를 관리할 때 유용합니다.
    

---

## 💻 코드 비교 예시

```cpp
#include <iostream>
#include <set>
#include <map>
#include <string>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    // 1. set: 중복 없는 명단 관리
    set<int> s;
    s.insert(10);
    s.insert(20);
    s.insert(10); // 중복 무시

    cout << "set count 10: " << s.count(10) << "\n"; // 1 출력

    // 2. map: 이름별 점수 관리
    map<string, int> m;
    m["kim"] = 90;
    m["lee"] = 80;
    m["kim"] = 100; // 값 갱신 (Key는 유일)

    cout << "kim's score: " << m["kim"] << "\n"; // 100 출력

    return 0;
}
```

## 🎓 편입 면접 대비 질문

> [!question] **"set에 저장된 값을 직접 수정할 수 있나요?"**
> 
> **Answer:** "아니요, 불가능합니다. `set`의 원소는 곧 정렬 기준이 되는 **Key**이기 때문에, 값을 직접 수정하면 트리의 정렬 구조가 망가집니다. 수정을 원한다면 기존 값을 **삭제(erase)하고 새로운 값을 삽입(insert)**해야 합니다. 이는 `map`의 Key도 마찬가지입니다."

> [!question] **"데이터 양이 방대할 때 두 컨테이너의 성능 차이는?"**
> 
> **Answer:** "두 컨테이너 모두 레드-블랙 트리를 사용하므로 삽입, 삭제, 탐색에 동일하게 **$O(\log N)$**의 시간이 걸립니다. 다만 `map`은 `pair` 객체를 저장하므로 메모리 사용량이 `set`보다 약간 더 많습니다."
