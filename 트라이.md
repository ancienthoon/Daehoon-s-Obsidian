# 🌲 트라이 (Trie) 자료구조

> [!abstract] **핵심 요약**
> 
> - **목적:** 수많은 문자열 중 특정 문자열이 있는지 빛의 속도로 찾기 위함.
>     
> - **특징:** 문자열의 각 문자를 노드로 만들어 연결합니다. 공통된 접두사(Prefix)를 공유하여 메모리를 절약하고 탐색 속도를 높입니다.
>     
> - **성능:** 문자열의 길이가 $L$일 때, 탐색 시간 복잡도는 **$O(L)$**입니다. (전체 문자열 개수 $N$과 무관!)
>     

---

## 1. 트라이의 작동 원리 (중학생 수준 비유)

영어 사전에서 `apple`을 찾는다고 가정해 봅시다.

1. 먼저 `a` 단원이 모여 있는 곳으로 갑니다.
    
2. 그다음 `p`가 나오는 곳을 찾습니다.
    
3. 또 `p`가 나오는 곳을 찾고, 순서대로 `l`, `e`를 찾습니다.
    

이때, `apple`과 `apply`는 앞의 `appl`까지는 똑같죠? 트라이는 이 **`appl`을 한 번만 저장**하고, 그 뒤에 `e`를 붙일지 `y`를 붙일지만 결정합니다. 즉, **뿌리에서 뻗어나가는 나무 가지**처럼 글자를 저장하는 방식입니다.

---

## 2. 왜 트라이를 쓰는가? (시간 복잡도 비교)

만약 10만 개의 단어가 저장된 목록에서 "hello"라는 단어가 있는지 찾으려면?

| **방식**        | **시간 복잡도**      | **설명**                             |
| ------------- | --------------- | ---------------------------------- |
| **단순 비교**     | $O(N \times L)$ | 10만 개($N$)를 하나씩 꺼내서 5글자($L$)씩 비교함. |
| **이진 탐색**     | $O(L \log N)$   | 정렬 후 이진 탐색을 해도 문자열 비교 시간이 추가됨.     |
| **트라이(Trie)** | **$O(L)$**      | 단어 개수와 상관없이 딱 **5번**만 이동하면 찾음!     |

## 3. C++ 실전 구현 구조

트라이는 각 노드가 **'다음 글자로 가는 포인터 배열'**을 가지고 있어야 합니다.

```cpp
#include <iostream>
#include <vector>

using namespace std;

// 알파벳 소문자 26개를 가정
const int alphabet_size = 26;

struct TrieNode {
    TrieNode* children[alphabet_size];
    bool is_end_of_word; // 단어의 끝인지 표시

    TrieNode() {
        is_end_of_word = false;
        for (int i = 0; i < alphabet_size; i++) {
            children[i] = nullptr;
        }
    }
};

// 단어 삽입 함수
void insert(TrieNode* root, string key) {
    TrieNode* curr = root;
    for (int i = 0; i < key.length(); i++) {
        int index = key[i] - 'a'; // 'a'를 0번 인덱스로 변환
        if (!curr->children[index]) {
            curr->children[index] = new TrieNode();
        }
        curr = curr->children[index];
    }
    curr->is_end_of_word = true; // 단어 완성 지점 표시
}
```

## 4. 트라이의 장단점

### ✅ 장점

- **압도적 속도:** 문자열 탐색에서 이보다 빠른 방법은 거의 없습니다.
    
- **접두사 탐색:** "ap"로 시작하는 모든 단어 찾기 같은 기능에 최적화되어 있습니다.
    

### ❌ 단점

- **메모리 과소비:** 각 노드마다 포인터 배열(C++ 기준 26개 $\times$ 8바이트 등)을 가져야 하므로, 메모리를 굉장히 많이 사용합니다. (백준 풀 때 메모리 제한을 잘 봐야 합니다.)
