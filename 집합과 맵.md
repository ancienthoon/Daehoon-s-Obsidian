

> [!info] **핵심 개념**
> 
> - **`set` (집합):** 중복을 허용하지 않고, 넣는 순서와 상관없이 자동으로 **정렬**되어 저장되는 주머니입니다.
>     
> - **`map` (맵):** **'열쇠(Key)'**와 **'값(Value)'**이 한 쌍으로 저장되는 사전입니다. 열쇠를 알면 값을 바로 찾을 수 있습니다.
>     

---

## 1. 집합 (set)

데이터의 존재 여부나 중복 제거가 목적일 때 사용합니다.

```cpp
#include <iostream>
#include <set>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    set<int> s;

    // 데이터 삽입 (중복은 자동으로 무시됨)
    s.insert(3);
    s.insert(1);
    s.insert(3); // 이미 있어서 무시됨
    s.insert(2);

    // 출력해보면 자동으로 오름차순 정렬되어 있음: 1 2 3
    for(int n : s) cout << n << " ";
    
    // 데이터 찾기
    if(s.find(2) != s.end()) cout << "\n2가 존재합니다.";

    return 0;
}
```

# 🗺️ C++ STL: map (연관 컨테이너)

> [!abstract] **핵심 정의**
> 
> - **Key-Value 구조:** 데이터를 '이름(Key)'과 '실제 값(Value)'의 쌍으로 묶어서 저장합니다.
>     
> - **자동 정렬:** 원소를 삽입하면 **Key를 기준으로 자동 오름차순 정렬**됩니다.
>     
> - **중복 불허:** 하나의 Key는 단 하나의 Value만 가질 수 있습니다. (Key는 유일해야 함)
>     
> - **자료구조:** 내부적으로 **레드-블랙 트리(Red-Black Tree)**로 구현되어 균형 잡힌 이진 탐색 트리의 형태를 띱니다.
>     

---

## 1. 시간 복잡도와 효율성

최근 학습하신 **'프로그램의 효율성'** 관점에서 분석한 `map`의 성능입니다.

|**작업**|**시간 복잡도**|**설명**|
|---|---|---|
|**삽입 (Insert)**|$O(\log n)$|트리 구조를 유지하며 자리를 찾아감|
|**삭제 (Erase)**|$O(\log n)$|삭제 후 트리의 균형을 재조정함|
|**탐색 (Search)**|$O(\log n)$|루트 노드부터 비교하며 내려감|

> [!tip] **언제 사용할까?**
> 
> 데이터의 개수가 많아도 $O(\log n)$의 성능을 보장하므로, 대량의 데이터에서 특정 키값을 빠르게 찾아야 할 때 유리합니다. 만약 정렬이 필요 없고 더 빠른 속도($O(1)$)를 원한다면 `unordered_map`을 고려할 수 있습니다.

---

## 2. 주요 멤버 함수

|**함수**|**기능**|
|---|---|
|`m[key] = val`|특정 키에 값을 저장하거나 수정함|
|`m.insert({k, v})`|키와 값의 쌍(pair)을 삽입함|
|`m.find(key)`|키를 찾아 반복자(iterator)를 반환 (없으면 `m.end()`)|
|`m.count(key)`|특정 키가 존재하는지 확인 (있으면 1, 없으면 0)|
|`m.erase(key)`|특정 키를 가진 요소를 삭제함|
|`m.clear()`|모든 요소를 삭제함|

---

## 💻 C++ 실전 코드 예시

```cpp
#include <iostream>
#include <map>
#include <string>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);

    // map 선언 <key_type, value_type>
    map<string, int> score;

    // 1. 데이터 삽입 및 수정
    score["kim"] = 90;
    score["lee"] = 85;
    score["park"] = 95;

    // 2. 데이터 탐색
    string target = "lee";
    if (score.find(target) != score.end()) {
        cout << target << "의 점수: " << score[target] << "\n";
    }

    // 3. 전체 순회 (Key 기준 오름차순으로 자동 정렬되어 출력됨)
    // it->first는 Key, it->second는 Value
    for (auto it = score.begin(); it != score.end(); ++it) {
        cout << it->first << " : " << it->second << "\n";
    }

    // 4. 데이터 삭제
    score.erase("park");

    return 0;
}
```

## 🎓 학습 포인트: 주의사항

> [!warning] **대괄호(`[]`) 연산자의 특징**
> 
> `m[key]`를 사용하여 데이터에 접근할 때, 만약 해당 키가 맵에 존재하지 않는다면 **자동으로 기본값을 가진 새로운 요소를 생성**해버립니다. 따라서 단순히 존재 여부만 확인하고 싶을 때는 `find()`나 `count()`를 사용하는 것이 메모리 낭비를 줄이는 방법입니다.


### 📊 std::set vs std::map 차이점

|**구분**|**std::set**|**std::map**|
|---|---|---|
|**데이터 구성**|**Key** (값 자체가 키)|**Key + Value** (쌍으로 저장)|
|**저장 형태**|`value`|`pair<const Key, Value>`|
|**접근 방식**|`find()`, 반복자(iterator)|**`[]` 연산자**, `find()`, `at()`|
|**중복 여부**|중복 불가|Key 중복 불가 (Value는 가능)|
|**시간 복잡도**|$O(\log N)$|$O(\log N)$|
|**주요 용도**|중복 제거, 존재 여부 확인|데이터 매핑, 사전(Dictionary)|

---

### 💡 핵심 요약

- **`set`**은 단순히 **"이 데이터가 주머니 안에 있는가?"**를 관리할 때 사용합니다.
    
- **`map`**은 **"이 이름(Key)에 해당하는 데이터(Value)는 무엇인가?"**를 관리할 때 사용합니다.
    
- 두 컨테이너 모두 데이터를 넣는 순간 **자동으로 정렬**된다는 공통점이 있습니다.
